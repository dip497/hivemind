Why hivemind

Running a coding agent in a bare terminal throws away everything around the work — which task it's on, what it changed, the session when you close the window. The moment you want a second agent on a second branch you're juggling tmux panes. hivemind makes the project the unit of work, not the terminal: one canvas per repo, several agents in parallel, each scoped to its own directory, branch and issues.

Agents in real terminals

Full PTY tiles for claude, codex, gemini, opencode — not chat boxes. Sessions survive the window and resume after a reboot.

Parallel by worktree & SSH

Bind a frame to a local repo, a git worktree, or a remote SSH host. Run agents side by side without conflicts.

Live diffs

A diff tile next to each terminal updates as the agent edits — split or unified, line comments you can send straight back.

Kanban PM you can cat

Issues, acceptance criteria, cycles and an activity log as plain markdown under .hivemind/. No DB, no API.

Agents update it over MCP

A small MCP server lets an agent pick up an issue, flip its state, tick acceptance criteria, and comment its progress.

Local-first

Everything on your machine. No telemetry, no cloud account, no SDK lock-in — agents use your existing CLI login.

FAQ

Is there a canvas or GUI for Claude Code?

Yes — hivemind is an infinite canvas for Claude Code and other AI coding agents. Each agent runs in a real terminal tile, alongside diffs, a file tree, an editor, an issues board and a browser, on one zoomable canvas per project.

How do I run multiple AI coding agents in parallel?

Bind each frame to a local repo, a git worktree, or a remote SSH host, then spawn an agent per frame. They work side by side — each scoped to its own directory, branch and issues — and you watch every diff update live.

Is it like tmux for AI agents?

Same idea, many agents on one screen, but with the structure agents need: a diff next to each terminal, a markdown issues board they update over MCP, worktrees as nested frames, and sessions that resume after a reboot.

Which agents are supported?

claude, codex, gemini and opencode out of the box; any terminal-native coding agent can be added.

Is it local-first and private?

Yes. Everything runs locally, the data is files on your disk, there's no telemetry, and agents use your existing CLI login — no extra API keys.
