# Changelog

All notable changes are documented here, in [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) format.
This project follows [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Each release is published to [GitHub Releases](https://github.com/dip497/hivemind/releases) with prebuilt artifacts (`hive-linux-x86_64`, `hivemind-<version>-x86_64.AppImage`).

## [Unreleased]

### Fixed

- **Tile names no longer fall back to "claude #N · auto" after a session reattach.** An agent tile's resolved name (claude's live OSC window-title task summary, e.g. "Fix AI Chatbot Service Request subject") was held in an ephemeral map that was wiped on every renderer mount and never persisted. A full app restart re-ran claude with `--resume`, which re-emitted the title, so names came back; but merely reattaching to a still-live daemon session replayed a serialized screen that omits the title OSC, so the name fell through to the generic spawn label. The last-known agent titles now persist in the canvas layout and seed on mount, so a reattach shows the real name immediately (a live OSC update still overrides it; a user rename still wins).

## [1.12.3] — 2026-07-09

### Fixed

- **Selecting a Layers row no longer shows a blue bar on its left edge.** The selected row carried an inset lavender-blue accent bar (`box-shadow` in `--color-brand`) on top of its neutral surface; it read as an unwanted blue block. The active row now uses the opaque neutral `--surface-4` fill plus a neutral hairline ring (no color) as its cue, and the click-to-focus ring suppression moved onto the button's own `outline-none focus-visible:outline-none` classes.

### Changed

- **Layers panel visual polish.** Workspace icons are now app-style rounded chips keyed to the frame color (repo = solid tile, worktree/remote = tinted chip with its branch/server glyph) so the three frame kinds read as one family instead of three differently-sized bare icons; nested worktrees and tiles now hang from a faint vertical guide rail so the hierarchy is legible at a glance; and the loose "Canvas" group icon matches the same chip family.
- **Opening a working diff no longer spikes renderer memory into the GBs on repos with a huge/generated/binary file.** A single large tracked or untracked blob (e.g. a 400 MB `index.scip` code-graph index, a lockfile, a bundle) was loaded in full as JS strings — twice, HEAD + working — and then line-diffed, exploding memory. `gitFileContents` now probes size first (`fs.stat` for working, `git cat-file -s` for HEAD/index) and refuses anything over 1.5 MB, so the blob never crosses the IPC bridge; the diff renders a "file too large to diff" placeholder instead.

## [1.12.2] — 2026-07-04

### Fixed

- **Selecting a Layers row no longer draws a blue focus-ring outline around it.** Clicking a row let the global `:focus-visible` outline fire; the row now prevents focus on click (`onMouseDown` preventDefault — keyboard Tab focus is unaffected), the `.hm-layers` focus override was hardened to `outline: none !important` for both `:focus` and `:focus-visible`, and the selected surface is an inline opaque neutral. Three independent guards so no stray blue block or ring survives, regardless of theme/glass state.

## [1.12.1] — 2026-07-04

### Fixed

- **The selected row in the Layers panel no longer shows a stray dark-blue block.** Under glass mode the selected-row background used `--color-bg4`, which glass redefines *translucent*, so the blue wallpaper bled through the row. The selected surface is now set inline to the opaque `--surface-4` neutral (glass never remaps it, and it doesn't depend on Tailwind arbitrary-class generation), with the brand accent bar as the selection cue. Also removed the brand-tinted focus fill so clicking a row never paints a blue block.

## [1.12.0] — 2026-07-04

### Fixed

- **A spawned agent's initial prompt now auto-submits — no more pressing Enter yourself.** The prompt was typed and an Enter sent 90ms later, but a freshly-booted claude TUI can drop that first Enter (it lands before the just-rendered input's paste-debounce settles), leaving the prompt typed-but-unsubmitted. Added a one-shot backstop Enter at 1.5s that fires ONLY if the agent is still idle (never submitted) — so a prompt that already went through never gets a stray second Enter.
- **A pinned tile is now a TRUE screen-fixed floating panel — constant position AND constant size, completely unaffected by canvas pan/zoom.** Rebuilt the feature: instead of keeping the pinned tile as a react-flow node and counter-translating it every pan frame (which held position but still *scaled with zoom* and could flash to its stale canvas spot on any node-array rebuild), the tile's content is now portaled out of react-flow's transformed viewport into a fixed, non-transformed screen-space layer (`#hm-pinned-layer`). Because that layer lives outside the canvas transform, the panel is inherently screen-fixed and constant-size. Drag it by the tile's own header (clamped inside the window), resize from the bottom-right corner; anchor + size persist across restarts. The **pin toggle is docked in every tile's own header/toolbar** next to close (terminals, agents, browser, diff, editor/workbench, issues), so pin↔unpin is one consistent button in one place — filled when pinned — rather than a chip floating detached above the tile; pinning also clamps the captured size so a zoomed-out tile doesn't open a postage-stamp panel. The tile keeps running through the pin (terminals reattach to their persistent daemon PTY); only a `<webview>` browser tile reloads its page once on pin — an inherent Chromium reparent limit.

### Added

- **Custom background & overlay media** — bring your own wallpaper and transparent foreground layers: pick any image/video for the canvas background, and **stack multiple** transparent `.webm`/`.gif`/`.png` overlays that play *over* your tiles (pointer-events-through, so they never block interaction). Files are copied into the app's data dir and served via a sandboxed `hivemedia://` protocol. In Appearance you can add/remove overlays and, per overlay, adjust opacity, fit, and **placement**: a size slider shrinks it below full-screen and a 3×3 grid parks it in any corner/edge/center (so a character can sit bottom-right *while* a banner sits top-center, etc.). Each overlay's file is keyed by a stable id so re-picking or adding one never disturbs the others. Off by default.
- **The installed agentic skills now document the full hive tool surface.** `hive-workflow` gained detailed coverage of `hive_send` (follow-up turns), `hive_send_keys` (answering a worker's native AskUserQuestion picker), the exact `hive_read` contract (`{ text, finalStatus }`, blocking-only), `supervise` + `hive_approve` (run an unattended fleet as the approval gatekeeper), `hive_list_tiles` (poll fleet status), `hive_open_review`, per-worker `model`, and the first-class `pi` runtime. `hive-work` now documents the cross-repo/link tools (`hive_link_issue`, `hive_move_issue`, `hive_list_workspaces`) and carries a full issue-tool reference. Embedded copies in `templates.ts` re-synced (base64 + raw) and round-trip-verified.

- **pi agents are now full orchestrators** — a pi worker can spawn/drive/supervise other agents: the injected pi extension registers native `hive_spawn_agent`/`hive_workflow`/`hive_read`/`hive_send`/`hive_list_tiles`/`hive_approve` tools (over the HCP socket, no MCP dependency) and brokers its own tool permissions to the supervisor (`tool_call` → `agent.await_approval`). The extension is now type-checked against pi's official `@earendil-works/pi-coding-agent` types.
- **Spawned agents now show a wire back to the agent that spawned them.** When an agent spawns a sub-agent (`hive_spawn_agent`) or a workflow spawns workers, a persistent dashed "parentage" line is drawn from parent → child on the canvas — for **every** spawn, including background workflow workers (previously a link was only drawn when `report` was on, so workflow workers appeared with nothing connecting them). It's visually distinct from the animated data-flow pipe (`hive_connect`): quiet, dashed, no traveling dot. The child still spawns into the **same frame** as its parent by default.
- **pi agents are now first-class** — spawned pi tiles report turn-completion, status, and their reply through hive (hive_read / auto-report / workflow now work with `agent: pi`), via an injected pi lifecycle-bridge extension (agent_start/message_end/agent_end → HCP), mirroring claude's Stop hook. Previously pi ran raw with screen-scrape-only status.
- **Claude model selection** — pick Opus or Sonnet per spawn (`hive_spawn_agent`/`hive_workflow` `model` param) or set a workspace default in Settings; threads to `claude --model`. claude-only.
- **`hive-workflow` skill installed on init** — `hivemind init --agentic` / workspace init now writes (and refreshes) the multi-agent orchestration skill, so agents know how to use `hive_workflow`.

## [1.11.0] — 2026-07-03

### Fixed

- **The `hive` CLI was broken in v1.10.6/v1.10.7 — it ran as raw Bun instead of hivemind.** The release workflow `strip`ped the `bun build --compile` binary to save ~1MB, but `strip` corrupts the JS-bundle trailer bun appends to the executable, so the binary silently degraded to the Bun runtime: `hive --version` printed Bun's version, `hive init` ran *Bun's* project init, and **every** subcommand (`new`, `list`, `task`, `ctl`) plus `hive mcp-stdio` (the MCP server agents call) failed. Removed the strip step. Re-run `hivemind upgrade` after this release to get a working `hive`. **This requires a new release** (the CLI asset must be rebuilt without strip).
- **`install.sh` / `hivemind upgrade` no longer wedges on a near-full disk.** An interrupted or killed upgrade (dropped network, closed terminal, SIGTERM) used to orphan a partial `hivemind.AppImage.$$.new` (up to ~115MB) because the temp-file `rm` only ran on curl's own in-process failure. On a near-full disk those orphans snowballed into ENOSPC and blocked every future upgrade. Now: a `trap … EXIT INT TERM` removes temp files on any exit, a startup sweep clears orphans left by prior killed runs, and a disk-space preflight fails early with a clear "low disk, need ~460MB" message instead of a cryptic truncated-download / extract error.

### Added

- **Pin a tile to keep it in view.** Hover any tile and click the pin badge (top-left) to make it a sticky HUD: it floats above other tiles and stays fixed on screen while you pan/zoom the canvas — pin a browser playing a video, then roam your code and it stays put. Click again to unpin (returns to its canvas spot). Pinned state persists across restarts. Pinning re-parents the tile without remounting it, so a pinned browser tile keeps playing (no reload). A pinned tile is clamped to the window so it can never scroll off-screen (a tile larger than the window keeps its header + pin control on screen). Note: v1 holds position through **pan**; a pinned tile still scales with zoom.

## [1.10.7] — 2026-07-02

### Fixed

- **Update checker no longer sticks on "Checking…" or falsely nags about an update.** Three compounding bugs: (1) the GitHub fetch had no timeout, so a stalled request pinned the UI on "Checking…" forever; (2) a failed check (offline / timeout / 403 rate-limit) was indistinguishable from "up to date" and could clobber a real result; (3) checking on every Settings open with no dedupe/throttle hammered the unauthenticated GitHub API (60 req/hr/IP) into rate-limit. Now: the fetch times out at 8s, results carry an explicit `ok` flag so only a *completed* check updates state/cache (a blip keeps the last known-good result), and checks are de-duped + throttled to once per 60s (the "Check now" button still forces an immediate check).

## [1.10.4] — 2026-07-02

### Fixed

- **`hivemind upgrade` no longer dies on `mv: cannot stat '…hivemind.AppImage.new'`.** `install.sh` downloaded to a FIXED temp name (`hive.new` / `hivemind.AppImage.new`) and then `mv`’d it unconditionally. Two upgrades running at once — a terminal `hivemind upgrade` racing the app’s in-app updater, or a double-run — shared that temp name, so one run’s `mv` stole the other’s file and the loser aborted with `mv: cannot stat`. A truncated/failed download (disk full, interrupt, rate-limit) hit the same `mv` with a cryptic error too. Now each run downloads to a **PID-unique** temp (`hive.$$.new` / `hivemind.AppImage.$$.new`), **verifies it’s non-empty before `mv`**, and cleans up + reports a clear reason on failure. Fetched from `main`, so it applies to the next `hivemind upgrade` without a release (`install.sh`).

## [1.10.3] — 2026-07-01

### Fixed

- **In-app update actually works now — live progress, a real restart, and clear success/failure.** “Update & restart” was half-wired: it ran the installer with `stdio:"inherit"` (invisible for a GUI launch), gave zero feedback for the ~30s download, then called `app.exit()` — it **quit instead of restarting**, and `relaunchApp` used a plain `app.relaunch()` that re-execs the current AppRun and never applies the staged build, so the update silently didn’t take and the “Update available” pill came back. Now: `runUpgrade` streams the installer’s output to a live toast, resolves with the real exit status, and does **not** quit; on success the UI shows “Update installed — restarting…” and relaunches **through the `~/.local/bin/hivemind` launcher**, which swaps in the staged build so the NEW version actually loads; on failure the app stays open with an error toast pointing at `hivemind upgrade`. Both the Settings “Update & restart” button and the canvas “Update available” pill now show a spinner + “Updating…” and go disabled while it runs (double-click-guarded). (`index.ts` `runUpgrade`/`relaunchApp`, `App.tsx` `useUpdateCheck`, preload `onUpdateProgress`, `canvas-islands.tsx`.)

## [1.10.2] — 2026-07-01

### Fixed

- **Diff viewer colors fixed — additions/deletions are vivid again, not washed out.** The `@pierre/diffs` CodeView's `*-color-override` CSS vars were being given translucent values (`rgba(...,0.14)`). Those vars feed the library's `--diffs-*-base`, a SOLID color the library uses directly for the header `+N`/`−N` counts, the classic `+`/`−` gutter markers, the `bars` indicator, and the inline word-diff span (the library layers its OWN alpha on top), and from which it DERIVES the row backgrounds via `color-mix`. So the 14%-opaque values made the counts/markers/word-highlights nearly invisible and the line tints washed out. Now bound to the solid oklch semantic tokens (`--color-ok` / `--color-err` / `--color-brand`) so the diff also stays in sync when the accent is recolored. Also dropped the dead `--diffs-selection-color-override` (documented but unconsumed by any library CSS rule) and re-tinted line selection to the brand accent (`pierre-codeview.ts`).

## [1.10.1] — 2026-07-01

### Fixed

- **Canvas + agent sessions now persist and resume no matter where you launch — including `$HOME`.** The canvas layout (which tiles/agents exist, their positions, and the per-tile session ids needed to `--resume`) was persisted in `localStorage` keyed by `repoPath`. Launching onto any folder that is **not** a git repo and **not** a `.hivemind` workspace — most commonly the default `$HOME` — resolved `repoPath` to `null`, so `saveLayout`/`loadLayout` no-op'd: the canvas was **never written**, every restart was a blank "Start with an agent" canvas, and agents never resumed even though the detached PTY daemon still held the live sessions. The canvas now persists under `repoPath ?? cwd` — keyed on the absolute folder path (never a shared `__global__` sentinel, so distinct folders never leak into each other), while a genuinely transient empty cwd (welcome/e2e bootstrap) still stays unpersisted (`Canvas.tsx`).
- **`$HOME` is now a valid workspace instead of a trap.** `findRoot` refused to resolve `~/.hivemind` at all (to stop a stray `hive init` at `$HOME` from hijacking every subfolder — the "only home gets selected" bug). That over-corrected: it also made a workspace initialised **at** `$HOME` dead-on-arrival. `findRoot` now resolves `$HOME`'s own `.hivemind` **when `$HOME` is the starting dir**, while still never letting a subfolder climb up into it (a `first`-iteration flag draws the line) — so home works as its own workspace with zero hijack risk. `home` is now an injectable arg for deterministic tests (`packages/hive-core/src/storage.ts`, `storage.test.ts` — +4 boundary tests, 49 pass).

## [1.10.0] — 2026-06-30

### Added

- **Pi (pi.dev) coding agent.** `pi` (Earendil's `@earendil-works/pi-coding-agent`) is now a first-class, enabled agent alongside Claude / Codex / opencode / Droid: pick it from the tool-island / frame launcher and it spawns a `pi` TUI tile (official Pi logo, theme-colored). Pi is a minimal harness with no hook system, so — like codex — it has no deterministic spawn-time signals: tile status comes from the screen-scrape detector (`detectPi`), and session resume across a daemon restart resolves the newest `~/.pi/agent/sessions` entry whose header `cwd` matches the tile cwd → `pi --session <id>`. Also wired through the CLI (`hive agent detect` finds `pi`, `--assignee pi` resolves as an agent) and the MCP `hive_spawn_agent` / `hive_workflow` runtimes list. Adds `main/pi-resume.ts`, `main/providers/pi.ts`; registers `piProvider` in the provider registry; adds `pi` to the renderer agent registry (`agents.tsx`).
- **Agent notifications now surface failures, respect preferences, and render as glass cards.** The notification system surfaces failures that previously died silently, gives you control over what reaches you, and shows every notice as a frosted-glass card. Four changes:
  - **Failure surfacing.** When an agent process dies while working (a crash, OOM-kill, or failed build that closed the shell) you now get a "failed" toast and native popup with the **exit code / signal in the body** (`137` = OOM, `143` = SIGTERM, …) — previously a non-zero exit only painted grey inline text you'd miss if you weren't looking. A clean `exit 0` still reads as "finished", not alarm-red.
  - **Notification preferences (Settings).** A new **Notifications** card in Settings: a master switch, per-kind mute (**Needs you** / **Finished** / **Failed**), independent surface toggles for **in-app toasts** vs. **native OS popups**, and a **Do-Not-Disturb** window (overnight wrap supported; needs-you always fires, since a blocked agent is urgent). Takes effect immediately — no relaunch.
  - **Glass toasts.** Agent toasts are now borderless frosted-glass cards (`backdrop-filter`, inset top-sheen + soft drop, no hairline border) with the agent label, the verb, the **exit-code detail**, the **frame/workspace** for context, a relative "just now / 12s ago" stamp, and a shrinking auto-dismiss line. A single accent (the app brand) tints the icon, verb, and dismiss line; the kind is encoded by the icon shape + verb, not a second color. Frosts against the live wallpaper under glass mode; the `sonner` action toasts (git push, issue CRUD) were restyled to match and the unused shadcn sonner wrapper was removed.
  - **Resilience signals.** A new `app:error` channel surfaces non-fatal background errors (starting with a stale PTY daemon that would otherwise silently break agent hook injection) as a toast instead of swallowing them, and a new app version triggers a one-shot **"Update available"** toast with an inline **Update & restart** action.

### Changed

- **Issue management UI refactored for a softer, more consistent feel.** The issue tracker (board, list, New issue dialog, and the issue detail peek) now shares one depth/elevation language: reusable `.hm-surface-*` / `.hm-card` / `.hm-popover` / `.hm-modal-*` CSS utilities (layered shadows + a 1px inner highlight) replace the patchwork of per-component shadows and borders. The IssuePeek and NewIssueModal popups now both use the Radix Dialog (matching backdrop, focus trap, and open/close animation) instead of one hand-rolled overlay + one Radix dialog. Board cards gained a state-color accent bar and a gentle hover lift; board columns and list groups got surface backgrounds, sticky soft headers, and count pills; FilterBar search has an inline clear button; and a shared `Popover` component unifies every dropdown (FilterBar, peek state select, assignee/label/parent pickers).
- **Color system re-aligned to Plane + Linear.** Researched Plane's (`makeplane/plane`, oklch dark theme) and Linear's design systems and re-based hivemind's entire palette on their tokens: cool blue-grey neutral surface ramp (oklch hue 230), Linear's lavender-blue `#5e6ad2` as the brand accent, Plane's green/amber/red semantic endpoints, and oklch-based shadows/highlights throughout for perceptually-correct alpha blending. All hex values in `:root` and `@theme` converted to oklch. Visually: backgrounds are slightly cooler/deeper, the brand is now lavender-blue (not indigo), state colors and label chips are richer and more scannable, and every surface stacks with the same depth language as Plane/Linear.

## [1.9.0] — 2026-06-29

### Added

- **Workbench is now an IDE — Explorer / Changes activity bar + in-tile diff review + commit.** The Workbench tile gained a VSCode-style left **activity rail**: **Explorer** (the full working tree — open any file to edit, with Pierre `⌘P` search + context menu + git-status decorations) and **Changes** (git-changed files with M/A/D badges). - **Delegated agents run autonomously by default.** When an agent spawns a worker via `hive_spawn_agent`/`hive_workflow` and doesn't pass an explicit `mode`, the worker now launches in autonomous mode (`--dangerously-skip-permissions`) — a delegated worker has no human at its tile, so inheriting the UI's human-gated default would hang it on the first permission prompt. An explicit `mode` still wins, and a `supervise`d worker keeps prompting (routed to the spawner for approval). Trust boundary unchanged: HCP is a `0600` same-uid socket, token-gated to hivemind-issued agents, depth-capped and rate-limited. User-clicked spawns still respect the toolbar mode.
- **Fullscreen (⤢) for terminal, editor, AND diff tiles — shared component, over the wallpaper.** A new shared `FullscreenShell` floats any tile on a **frosted-glass panel (same content-tint + blur as the canvas tiles) over a clean copy of the live wallpaper/video** — not the canvas + other tiles bleeding through; **Esc** exits, and the header shows font `A−/A+` (density). Terminal + editor reparent their live node (xterm element / CodeMirror view — PTY/scrollback/undo preserved); diff remounts the Pierre CodeView inside the panel. The terminal grid grows at 100% zoom (crisp, no scaling).
- **Terminal scale + font controls, hover-revealed.** The terminal header now reveals, on hover, two separate controls: **font size** (`A−/A+`, density only) and **whole-tile scale** (`−/+`, grows the node box AND font in proportion — crisp, nothing zooms).
- **"Update available" notification + Settings/About panel.** On launch and every few hours the main process checks the latest GitHub release (`api.github.com/repos/dip497/hivemind/releases/latest`) and compares it to the running version. When a newer release exists, an "Update available" pill appears beside the Theme button (and a dot on the new gear button); clicking it runs the official `install.sh` upgrade flow and quits so the new binary takes over on next launch. A failed check (offline / rate-limited) shows nothing. The new **Settings & About** drawer (gear button, frosted `.hm-island` like the theme drawer) shows the app name, version, GitHub repo link, MIT license, and update status with a one-click "Update & restart" / "Check now". The update check runs in main (renderer CSP blocks the github.com fetch); the last result is cached in `localStorage` so the badge survives a reload.

### Fixed

- **Wallpaper imports no longer leak disk space.** Picking a video/photo wallpaper copied the file into `userData/wallpapers/` but never removed the previously-imported ones — every wallpaper you ever tried piled up (hundreds of MB of orphaned `.mp4`s). Import now prunes older files of the same kind (only one video + one image can be active), so the dir stays at ~one of each. Re-select your wallpaper once to reclaim the existing orphans.
- **Plan-review hook now logs why it falls open.** The injected `PreToolUse(ExitPlanMode)` hook is fail-open (any error → native prompt, silently). It now appends the reason to `userData/plan-hook.log` (invocation + connect/close errors) so a non-working in-canvas plan review is diagnosable instead of silent.

### Changed

- **Clearer `hive_workflow` / `hive_spawn_agent` tool descriptions.** Documented the autonomous spawn default, per-worker `status` semantics (`turn`/`timeout`/`error`), that a pipeline stops at the first failed stage and returns the partial run, the practical `items` ceiling (rate-gated), and that non-claude runtimes must be installed.

## [1.8.4] — 2026-06-26

### Fixed

- **Terminal text is now crisp instead of blurry.** Two causes: (1) `allowTransparency` was set unconditionally for the glass feature, and it disables subpixel-antialiased text — so it's now only enabled when Frost-tile-content is actually on; opaque terminals keep subpixel AA. (2) Agent (claude) terminals were pinned to xterm's WebGL renderer, which is grayscale-only (softer); they now use the **DOM renderer whenever idle or selected** (subpixel, sharp like a native terminal) and drop to WebGL **only while actively streaming** — so a multi-agent fan-out still can't spike the renderer, and renderer swaps happen only on unselected tiles (no visible flicker). Net: idle terminals are as sharp as a native GNOME terminal. (For maximum sharpness keep Frost-tile-content off — transparent text physically cannot use subpixel antialiasing.)

## [1.8.3] — 2026-06-25

### Fixed

- **Glass mode no longer spikes the renderer (~227% → ~9% CPU).** Frost tile content was forcing every terminal onto the slow DOM renderer (on the wrong assumption that WebGL ignores transparency — it doesn't), so streaming agents repainted as DOM nodes. Terminals now stay on WebGL with a transparent background; verified live, idle renderer CPU dropped from ~227% to ~9%.
- **Frosted terminal "gap" removed.** The terminal's tint now lives on the tile root (like every other tile) with a fully-transparent xterm, so the whole body — including the inner padding band — reads as one uniform tint instead of a shaded frame around the content.
- **HEVC/H.265 video wallpapers no longer show a black screen.** A clip that can't decode now falls back to the gradient wallpaper, and HEVC hardware decode is enabled via VAAPI (`PlatformHEVCDecoderSupport` + `VaapiVideoDecoder`) where the GPU supports it. (H.264 `.mp4`/`.webm` always worked.)

### Changed

- Text selection is now a neutral translucent white across terminal/editor/diff (was the brand accent).
- Cleaner selection + resize affordances: a selected tile shows a small accent clip above its top edge instead of a full outline, and resize handles stay hidden until you hover a corner/edge.
- Clearer git-compare **Diff** tool-island icon.
- Frosted panels drop their blur over **video** wallpapers (the tint stays) to avoid a per-frame re-blur cost; gradient/photo wallpapers keep their blur.

### Fixed

- **A tile can no longer be stuck on "working" when it isn't.** Every "working" claim — whether from a hook turn (`liveTurn`) or the screen-scrape — is now gated on *recent pty output*: a genuinely-working agent streams output (animated spinner / elapsed-seconds timer / tokens, ~every second), while a finished, Esc-interrupted, killed, or restart-replayed tile is **quiet**. So a "working" with no output for ~8s is treated as stale and reads **idle**. This is the universal self-heal that the prior scrape-vs-hook reconciliation couldn't cover: it doesn't depend on the scrape detecting an idle prompt (which can itself freeze on a replayed buffer) or on the `Stop` hook firing (which an Esc-interrupt skips). Background subagents (event-driven, legitimately quiet) and needs-you prompts are exempt. The bus learns liveness from a new `noteOutput` (called on every pty data event) and re-evaluates each poll tick via `revalidate`. A staleness-decay is flagged **synthetic** — the awareness layer updates the status but does NOT treat it as a completion (no toast, no OS notification, no done-unseen pulse, no canvas re-render), so correcting many stuck tiles at once can't flood the screen with "finished" toasts or jank the app. Window is 15s (long enough that a working agent's normal output cadence — even batched under load — never false-decays).
- **Spawned-agent tiles (incl. workflow workers) no longer stick on "working" after they finish.** The hook-driven turn status was keyed wrong across the id namespaces: the `UserPromptSubmit`/`Stop`/`Notification`/subagent hooks report the **pty id** (`hm:<bare>` = `HIVEMIND_TILE`), but the renderer status bus keys by the **bare** tile id — so for any HCP-spawned tile the deterministic "idle" landed under `hm:tile-…` and never reached the tile, which fell back to the screen-scrape (and droid's scrape can read a finished screen as still-working). The turn-tracker normalizes via the pty id, so *gathering* worked while the *status pill* lied. Main now `toBareId()`-normalizes the tile id before every renderer status push (turn / subagent / notification), matching the bus + the `onExit` cleanup. This also makes claude's intended hook-driven status actually take effect (it had been silently running on the scrape). `recordTurn` stays on the pty id (the tracker's key), so gather is unaffected.
- **Workflow workers no longer steal focus or spam "finished" notifications.** A tile spawned by `hive_workflow` (or any HCP spawn with `report:false`) is a background worker the orchestrator gathers in bulk — so it now spawns WITHOUT grabbing keyboard selection or centering the viewport on itself (you keep typing where you were while a fan-out runs), and it no longer raises a per-worker "… finished" toast / OS-notification (N workers → N pings was noise). A "needs you" (blocked/permission) signal is still surfaced, since an unattended blocked worker genuinely needs a human. Threaded via a `background` flag on the spawn (`report === false`) → `placeInFrame` skips select/focus and a small `worker-tiles` registry gates the notification. Normal `hive_spawn_agent` (report:true) is unchanged — it still focuses + pings.
- **Non-claude agents (droid / codex / opencode) now receive their spawned prompt.** A tile spawned with a prompt (via "Work on this", `hive_spawn_agent`, or `hive_workflow`) queues that prompt and types it in the moment the agent first reaches its input — but the delivery was gated to claude *identity* only, so a droid/codex/opencode worker spawned, sat at its prompt, and never received the message ("message not going"). It's now delivered for **any** known agent (plain shells have no queued work, so they're unaffected). This is what made droid workflow workers spawn-but-do-nothing.
- **Agent tiles no longer get stuck reading "working" forever after they finish.** A tile reads "working" while it has in-flight Task subagents (so background agents the main loop has moved past still register). But the `SubagentStart`/`SubagentStop` hooks aren't a guaranteed matched pair — a stop edge is lost when a subagent errors, the turn is interrupted (ESC, which also suppresses the main `Stop`), the session is compacted, or the process dies. A lost stop left the in-flight set non-empty permanently, pinning the tile "working" (it outranks the hook-driven idle). We can't just drain the set on `Stop` (a `run_in_background` subagent legitimately keeps running past it), so a **watchdog** now reaps the set if no subagent edge arrives within a quiet grace window (120s) — every real edge pushes the deadline out, so a genuinely-active background population is never reaped, while a stuck set self-heals to idle. Reaps are logged to `render-diag.log`. Adds `main/hcp/subagent-reaper.ts`.
- **Agent tiles no longer get stuck "working" when a turn ends without a `Stop` hook.** The other half of the stuck-working problem: the hook-driven turn state (`UserPromptSubmit` → working, `Stop` → idle) hard-won over the screen scrape, so when `Stop` never fired — the turn was interrupted (Esc), the agent was killed, or the app restarted mid-turn — the state stuck at "working" with nothing able to correct it. A **stabilized scraped `idle`** (the agent sitting back at its prompt) now self-heals a stale "working". The `idle` hook-state still overrides a spinner false-"working", and a live background subagent still keeps the tile "working" (subagent-busy outranks this). - **The canvas no longer re-renders every second while an agent works** (the cause of the mouse cursor flickering between shapes ~every second, and a lot of the lag). The agent-awareness hook re-rendered the entire Canvas — and thus the whole react-flow tree — on every status-bus event, and a working agent re-publishes its status on each ~1s poll tick. Now `hive_spawn_agent` defaults to `report: true`: the moment the worker finishes a turn, its **clean reply** (read from Claude Code's session-transcript JSONL via the injected `Stop` hook — never screen-scraped) is delivered straight into the parent agent's session as a `[hive] from <tileId>:` message, and a **visible reporting edge** is drawn from worker → parent on the canvas. So you can fire-and-forget a delegated task and keep working; the answer arrives in your inbox when it's done. This reuses the existing agent-pipe machinery (auto-connecting the worker to its parent). `hive_read` still exists for synchronous use but now returns clean transcript text, and on timeout reports `finalStatus:"timeout"` ("agent still working") instead of terminal garbage. `report:false` opts out for a worker you'd rather poll. (Validated against Claude Code's hook model: `Stop` is the reliable per-turn signal, the transcript JSONL keyed by our injected `--session-id` is the clean source, and PTY stdin is the only supported way to hand a running interactive agent new input — this complements native Task subagents rather than reinventing them.)
- **Diff tile: new "unpushed" review mode** — review code you've committed locally but not yet pushed, beside the existing "working" and "branch" tabs. It diffs `@{upstream}...HEAD` (the net diff of local commits ahead of the branch's remote tracking ref — exactly what you'd push), falling back to the default branch when the branch was never pushed (every local commit is then unpushed). The tab shows the ahead-commit count as a badge and disables itself when the branch is tracked and fully pushed. Built on a new additive `{ kind: "unpushed"; base? }` `DiffScope` and a `resolveUnpushedBase()` resolver mirroring the branch-base logic; the mode toggle is now a data-driven list so future diff sources (a single commit, an arbitrary range, "since tag") slot in with one entry + one scope variant.

### Fixed

- **Trackpad scroll no longer "leaks" into a canvas pan at the end of a flick.** Two separate leaks: (1) native scroll-chaining at the inner element's edge — fixed with `overscroll-behavior: contain`; and (2) our *custom* wheel-pan, which `overscroll-behavior` can't govern (it's `preventDefault` + manual viewport math, not native scroll). A trackpad keeps emitting wheel events after your fingers lift (momentum); when the terminal hit the bottom of its scrollback mid-flick, that inertia was read as "nothing to scroll → pan the board", so the content scrolled and then the whole panel lurched. The rule is now **position-independent**: if a tile contains scrollable content (a terminal, or any overflow container), the wheel belongs to that content and *never* pans — at the edge it simply stops. Because there's no scroll↔pan transition at the boundary, the momentum tail has nothing to flip into. Only a tile with no scroller at all pans on wheel-over; panning a scrollable tile is via space/middle-drag or the empty canvas, as before. (Researched against MDN wheel-event docs + the `lethargy` inertia-detection approach; the position-independent rule is simpler and fully eliminates the transition.)
- **Clicking a tile in the Layers panel now focuses it exactly like clicking it on the canvas.** The panel used the older fit-to-screen zoom-out, so navigating to a terminal from the panel landed at a different zoom/position than selecting it directly. It now mirrors the canvas-click decision: text tiles (terminal/shell/diff/editor) snap to 100% with the content-corner clamp; other tiles get the framed focus.
- **Focusing a terminal no longer clips its left edge off-screen.** Selecting a text tile snaps it to exactly 100% (xterm/diff/editor are only pixel-accurate at 1:1), but the old path zoomed to 1 around the *viewport* centre — if the tile wasn't already centred (or was wider than the window) it grew with its left columns (the prompt + line starts) hanging off the left, unreadable. Focus now recentres on the tile, and when the tile is bigger than the viewport it anchors the content corner (left/top) instead of centring, so the readable part is always on screen. (This also unifies two camera controllers that previously fought — the fit-to-tile focus and the snap-to-100% — into one move.)
- **The PTY daemon no longer dies when you close the window on GNOME/systemd desktops.** The daemon is spawned `detached` (its own session, via `setsid`) so it should outlive the window — but GNOME launches the app inside a transient `app-gnome-<name>-<pid>.scope` cgroup whose default `KillMode=control-group` terminates *every* process in the cgroup on close, and `setsid` escapes the process group/terminal but **not** the cgroup. So on every close the daemon (and all your terminal sessions) was killed, defeating the whole point of persistence. The daemon is now registered in its own transient scope via `systemd-run --user --scope` (a `run-*.scope` outside the app cgroup) when we detect we're inside a killable app scope — verified to land in a separate cgroup that survives the window closing. Falls back to the plain detached spawn on non-systemd/non-app-scope environments (containers, plain login sessions, other desktops).
- **Plan review now opens in the agent's OWN frame/project.** The PreToolUse hook hands the renderer the agent's `HIVEMIND_TILE`, which is the PTY id (`hm:<tileId>`), but `openPlanReview` did a raw frame lookup on it without stripping the `hm:` prefix — so it always missed and the review tile landed in the active/first frame. If you planned in one project (manageark) while another (motadata-itsm-server) was active, the review opened over the wrong one. The prefix is now stripped (same seam as `hive_spawn_agent`), so the plan renders beside the agent that produced it.
- **A tile that fails on a stale lazy chunk now offers a real "Reload" instead of a dead "Retry".** After a rebuild, an already-open window references old `assets/<name>-<hash>.js` files; a lazy import (e.g. the plan-review markdown preview) then 404s with "Failed to fetch dynamically imported module" and the tile error boundary's *Retry* just re-requested the same dead hash. The boundary now recognises stale-chunk / `ChunkLoadError` failures and shows a **Reload** button (reloads the window to pick up the fresh build; PTY sessions survive in the daemon).
- **Plan-review tiles now show up in the Layers panel with a "review" status.** A plan-review tile is a live agent blocked on your decision, but it was excluded from the panel — so there was no way to navigate to it and no status indicator. It now appears under its frame with a `▤` glyph and a `review` status, surfaces in the "needs you" filter (a blocked agent is waiting), and clicking it focuses the tile like any other layer.

- **Terminal/tile scroll no longer leaks into a canvas pan.** Scrolling a focused tile (claude, diff, editor, …) used to scroll its content for one notch, then "escape" and pan the whole board once the inner content hit its scroll boundary — which a terminal hits constantly. A focused tile now carries the `nowheel` class so react-flow ignores the wheel over it entirely; the content scrolls and the canvas never pans. (Unfocused tiles still pan on wheel-over as before.) Also fixed a wheel-listener leak where the cleanup's capture flag didn't match the registration. **Follow-up:** the leak had a *second* cause unrelated to react-flow — at the terminal's scroll boundary the wheel's native overscroll chained up to the document and shifted the whole panel. The page root is now `overflow:hidden; overscroll-behavior:none` (it's a fixed-viewport canvas, nothing at page level should scroll) and `.xterm-viewport` is `overscroll-behavior:contain`, so terminal scroll stops at its own boundary. **And the wheel is now scroll-aware:** every tile carries `nowheel` (react-flow never pans/zooms over a tile), and the tile's own wheel handler checks whether the element under the cursor still has scrollback/scroll room in the wheel's direction — if it does, that element scrolls and the canvas does NOT pan; only when there's nothing left to scroll internally (or the tile is at its scroll boundary, or has no inner scroll) does the wheel pan the board. So hovering an *unselected* terminal with scrollback now scrolls the terminal instead of panning the canvas.
- **HCP hardening pass (council-driven).** A 5-agent review (reviewer / architect / code-quality / maintainer / devil's-advocate, run *through* the control plane) surfaced gaps that were then closed: the bare↔`hm:` tile-id mapping is centralized in one `shared/tile-id.ts` seam (was reimplemented in 4+ places); **pipe forwarding now actually fires** (both the `dests()` lookup *and* the write step convert through the `hm:` pty namespace — a verification agent caught that the first pass fixed only the lookup); **agent pipes refuse cycles** at connect time (an `A→B→A` ring would otherwise pump forever once forwarding is live); **dead-tile sends fail loud** (`agent.send` to a non-existent tile now throws `TILE_NOT_FOUND` instead of reporting success); the **spawn-depth gate is enforced** (`DEPTH_EXCEEDED` past depth 3, bounding agent-spawns-agent recursion — previously specified but never checked); **`hive_read`'s `timeout_ms` reaches the wire client** (long reads no longer die at the 130s client default); and the token-less `turn` event is **constrained to real `~/.claude/**.jsonl` transcript paths** (a forged event can't make a read return an arbitrary file's contents). A follow-up P1 pass: the transcript is **tail-read** (bounded 256 KiB window) instead of full-read every turn (was O(n²) over a session); closing a tile now **forgets its pipes + prunes all per-tile control state** (no leaks, no forwarding into a dead tile); and the prompt-submit delay is a single `SUBMIT_DELAY_MS` constant. Remaining P1/P2 items are catalogued in `docs/reviews/hcp-fix-plan.md`.
- **`hive_read` / `agent.send` now address the right tile.** The control surface uses bare tile ids (`tile-…`) but the pty, output recorder, and Stop-hook turn reports are all keyed by the pty id (`hm:tile-…`). `agent.send`/`agent.read`/`agent.stream` now map between them, so a send reaches the agent and a read sees the turn the Stop hook reported (previously `hive_read` timed out even though the worker had replied).
- **Prompts delivered to a spawned/sent agent now actually submit.** The text + Enter were written in a single chunk, which claude's TUI dropped (the prompt sat unsubmitted in the composer). All delivery paths — the queued spawn prompt, `hivemind:send-to-claude`, HCP `agent.send`, and pipe-forwarding — now type the text and press Enter (`\r`) as a *separate* keystroke a tick later (the tmux `send-keys` pattern).
- **`hive_spawn_agent` now lands the worker in the SPAWNING agent's frame**, not a random/active one. The control plane injects the agent's own tile id (`HIVEMIND_TILE`) into its env, the MCP passes it as the caller, and the renderer defaults the new tile into the caller's frame (stripping the `hm:` pty-scope prefix to match; explicit `frame` still wins).

### Added

- **Clickable file paths in the terminal.** Paths in terminal output — `/abs/file.html`, `./src/x.ts`, `~/notes.md`, `file://…`, and `path:line:col` from stack traces — are now links: clicking opens the file in the OS default app (`.html` → browser, source → editor, image → viewer, dir → file manager) via `shell.openPath`. - **Hivemind Control Plane (HCP) — a programmable multi-agent canvas.** A local control plane lets an agent inside a hivemind tile (or a CLI/script) drive the running app: spawn sibling agents on the canvas, converse with them, wire them together, and request human review. The hive MCP gains canvas tools — `hive_spawn_agent` (launch an agent tile + hand it a task → returns a tileId), `hive_send`, `hive_read` (block until it finishes a turn and return its reply), `hive_list_tiles` (with live agent status), `hive_focus`, `hive_close_tile`, `hive_connect`/`hive_disconnect` (pipe one agent's replies into another's input — chain workers), and `hive_open_review` (open a plan in the visual review tile and block for human approve/deny). The MCP's issue tools keep working file-only when the app isn't running. Design validated against tttt / tmux-orchestrator / Google A2A / Claude Code agent-teams — hivemind's angle is making the agent mesh **visible, spatial, and cross-tool**, complementing (not duplicating) Claude Code's native subagents. (Deferred: per-child spawn-depth limits and a remote/devcontainer HTTP adapter.)

- **Plan review** — when a `claude` agent finishes planning and hands off (ExitPlanMode), hivemind opens a native **Plan review** tile on the canvas beside that agent, instead of the agent silently proceeding. ### Fixed (review pass)
- **`readConfig` self-heal race + non-atomic write.** Multiple IPC handlers (`createIssue`, `listIssues`, `resolveProject`, …) hit `readConfig` in parallel on first launch; when the file was invalid, every caller raced into `writeConfig` and a bare `fs.writeFile` could half-truncate. Now an in-process `Map<root, Promise<Config>>` coalesces concurrent reads to share one repair pass, and `writeConfig` writes to `config.yaml.<pid>.<ts>.tmp` then `fs.rename` (atomic). New test pins 20-way concurrent readConfig + verifies the file stays valid.
- **Editor `diffMode` map no longer leaks.** Closed tabs left their `diffMode[path]` entry behind; opening a stale path resurrected `true` and triggered an unwanted `gitFileContents` fetch. Tab-prune effect now drops `diffMode` entries alongside `meta` + `buffers`.
- **Activity timestamps no longer churn the file on first update.** `ActivityEntry` now carries `rawAt` (the on-disk form); serializer round-trips `rawAt ?? at`, so legacy `YYYY-MM-DD HH:MM` rows stay verbatim until a NEW entry is appended (the new one uses ISO-Z). Previously, the first `updateIssue` after upgrade rewrote every activity line and produced huge noisy diffs. New test pins zero churn.
- Dropped unused `getOriginalDoc` re-export in `EditorTile.tsx` (dead-import workaround).
- Editor + diff tile defaults now clamped to `window.innerWidth − 80` so 1400×900 doesn't spawn off-screen on 1366px laptop screens.

### Note
- **Frame selection: click the header to select.** With `pointer-events: none` on the frame body (the pan fix), clicking the dashed body region no longer selects the frame. Header bar remains the selection / drag handle.

### Fixed
- **No more huge gap when spawning a tile in a frame.** The collision placement used a fixed column step of the widest-possible default tile (1400+px), so a new tile jumped a full tile-width to the right even when the existing tiles were narrower — leaving a big empty band. Now the new tile packs snug to the right of the rightmost existing tile's actual edge (+24px gap), top-aligned. Frame grows then `fitFrame` tightens.
- **Opening a tile in a frame no longer stacks it on top of an existing one.** `placeInFrame` used a fixed 1240×860 grid with `Math.round` bucketing; once tile defaults grew to 1400×900 a wide tile straddled two buckets, occupancy mis-detected, and the new tile landed on top of a sibling. Replaced with real rectangle collision: gather the actual boxes of every tile already in the frame, scan candidate slots left→right / top→down, place in the first that overlaps nothing, grow the frame to contain it, then `fitFrame` snaps it tight. New tile lands beside the others.
- **Frame `+` / color popovers render on top instead of behind tiles.** Frame nodes sit at zIndex ≤ 90 (below tiles); a child popover can't escape its parent's stacking context, so the "open in zone" menu and the color picker rendered BEHIND any tile inside the frame. Both now portal to `document.body` with fixed positioning anchored under their trigger button (z-9999) via a shared `AnchoredMenu` component with a click-away catcher.

### Changed
- **Frames now auto-size to their contents (architecture change).** Frame geometry is *derived* reactively from the bounding box of the tiles inside, via **explicit membership** (a persisted `frameOf` tile→frame map) — NOT geometry. A tile entering a frame grows it; moving a tile reflows it; the last tile leaving collapses the frame to a small labeled placeholder (still a visible, droppable workspace zone). Why explicit membership: a geometry-derived rule deadlocks — a freshly-spawned big tile whose center sits outside a collapsed placeholder frame would never be claimed, so the frame could never grow to it. `frameOf` breaks that loop entirely; geometry maps to membership in exactly one place (a tile's drop in `onNodeDragStop`), never in a render-reactive path.
  - **Membership set on drop/spawn**: a tile joins the frame whose rect contains its CENTER on drop, or becomes loose if dropped outside every frame. Spawning into a frame records membership directly.
  - **`extent: "parent"` dropped** — tiles are free to leave a frame (required for collapse-on-leave; `extent` previously clamped a parented tile *inside* so it could never be dragged out).
  - **Frames are no longer manually resizable** — `NodeResizer` + the ⤢ fit button removed; geometry is auto. Breathing room = the `FRAME_PAD` constant.
  - **Dragging a frame carries its tiles** — every `frameOf`-member's absolute position is translated by the frame delta on drag-stop.
  - **Migration**: old layouts (no `frameOf`) seed membership once from geometry on load.
  - Loop-safe: the auto-fit effect reads `frameOf` (not geometry) for membership, computes against `prev` frames in the updater, 2px dead-band, `frames` not a dependency. Fires only at human-action frequency.
  - `apps/desktop/src/renderer/src/Canvas.tsx` (`frameOf` state + persistence + migration, auto-fit effect, `mkTile`/`frameTiles`/`placeInFrame`/frame-drag all keyed on `frameOf`), `FrameNode.tsx` (NodeResizer + fit button removed).
- **Editor: inline diff toggle per tab (`@codemirror/merge` `unifiedMergeView`).** Each open tab now has a `⇄` button (active tab in the tab bar, plus a duplicate in the standalone tile header). Clicking it overlays the buffer's diff against `HEAD:<path>` in the SAME editor — deletions as widget rows, additions inline, per-chunk accept/reject controls. Toggle off returns to plain edit mode without rebuilding state (CodeMirror `Compartment.reconfigure`). Standalone `DiffTile` stays for browsing branch/commit diffs (where there's no editable buffer). Research: marimo-team/codemirror-ai, aziis98 review-tool walkthrough, Sourcegraph blob view — all use the same `unifiedMergeView` extension pattern instead of a separate diff view class.

### Fixed
- **Editor now focuses freshly-opened tabs.** Clicking a 2nd / 3rd file in the file tree appended a tab but kept focus on the first — `EditorTile`'s `active`-keepalive only re-targeted when the current tab was *removed*. Now we diff `tabs` against the previous render and switch `active` to any newly-added path (last wins if several land in one render).
- **Spawning a tile now always pans the viewport to it.** Previously the focus-on-spawn was gated on `nodes.length > 1`, so the very first tile of a session (or a tile that landed off the default viewport inside a frame) got selected but never centered — felt like "spawn did nothing." Applies to claude, extra terminals, shell, editor, diff, and issues — including the `+` menu inside frames.
- **Larger default sizes for Editor and Diff tiles.** Bumped from 760×520 / 720×460 → 1400×900, in line with the claude/terminal default. The old sizes truncated file lists and diff hunks badly on first open.
- **`FocusOnTile` no longer gates on DOM-measured dimensions.** Previously `fitView` was only called once xyflow had measured the new node's width/height — but a culled node (`onlyRenderVisibleElements` cuts off-screen nodes from the DOM) never gets measured, so the retry loop expired silently and the very first spawn of a session never centered. Now we call `fitView` immediately; xyflow's internal `batchContext.nodeQueue` queues the fit-view until the next node batch flushes (verified against `@xyflow/react/hooks/useReactFlow.ts`).
- **Diff/FileTree explicitly respect `.gitignore`.** Added `--ignored=no` to `git status` (already the default, made explicit so flag drift can't silently expose ignored files). `git ls-files --exclude-standard` already does the same for the file tree. **Note:** a file that was COMMITTED before being added to `.gitignore` will still show — that's how git works; `git rm --cached <path>` removes it from tracking.
- **Activity timestamps no longer drift by your timezone offset.** Activity rows were stored as `YYYY-MM-DD HH:MM` (UTC, but with no `Z` suffix). The renderer's `new Date(str)` parses that format as **local time** — so users in IST (+5:30) saw every newly-created activity row as "5h ago" instantly. Now stored as full ISO with `Z`. Parser still accepts the legacy format and normalizes it to ISO on read — old issue files auto-migrate on next write.
- **Pan (palm) works inside frames again.** xyflow auto-tags every draggable node with the internal `.nopan` class (see `@xyflow/system` d3-zoom filter), which blocks pan started from inside the node. Frames cover huge regions so pan / right-drag / Space+left-drag / rubber-band selection all died inside them. Fix: `.react-flow__node-frame { pointer-events: none }` so the browser routes the gesture to the `.react-flow__pane` container; header bar + resize handles opt back in via `pointer-events: auto`. Selecting a frame is via its header bar.
- **`readConfig` now self-heals broken `.hivemind/config.yaml`.** Configs missing `prefix` (early-init files, hand-edited workspaces) used to hard-throw `bad_config` from every IPC call — `createIssue`, `listIssues`, every UI action — surfacing as a raw stack trace. Now we fill defaults (prefix derived from repo dir name, `next_id: 1`, `agents: {}`) and rewrite the file once. Same path handles a missing config file entirely. Tests in `packages/hive-core/src/storage.test.ts` pin the behavior.
- **install.sh: AppImage now extracted instead of FUSE-mounted.** Ubuntu 22.04+ ships without `libfuse2`, which made the AppImage fail with `dlopen(): error loading libfuse.so.2` on every fresh install. Installer now runs `--appimage-extract` and symlinks the bundled `AppRun` — no FUSE, no sudo, works on all glibc Linux.

### Removed
- **Cycles** — dead feature ripped out: CLI `hive cycle` command, MCP `hive_list_cycles` tool, `Cycle` types, `listCycles`/`readCycle`/`writeCycle` storage funcs, `cycle` field on issue frontmatter, IPC `listCycles`, renderer `useCycles` hook. Never had any UI consuming it. Breaking change for any external script using `hive cycle …`.

## [0.0.1] — 2026-05-27

### Added
- First public OSS release.
- **CLI** (`hive`) — `init`, `init --agentic`, `new`, `mcp-stdio`, and friends. Single-binary distribution via `bun build --compile`.
- **Desktop app** — Electron + xyflow infinite canvas with terminal / diff / file-tree / editor / issues tiles, frame-as-workspace zones, persistent PTY daemon, Mosh-style replay (headless xterm + SerializeAddon), disk-backed snapshots, claude `--session-id` binding for deterministic resume across reboot.
- **MCP server** (`@hivemind/mcp`) — 9-tool stdio server wrapping `@hivemind/core` so claude can `get_issue`, `set_state`, `add_comment`, `mark_acceptance`, etc.
- **Templates** — per-workspace `.mcp.json` + `CLAUDE.md` + `.claude/skills/hive-work/SKILL.md` copied by `hive init --agentic`.
- **install.sh** — single script for both fresh install and in-place upgrade. Downloads prebuilt binaries from GitHub Releases by default; `--dev` flag clones and builds from source.
- **GitHub Actions** — `release.yml` (tag-driven build + publish on `v*.*.*`), `ci.yml` (typecheck + build + unit tests on every push / PR).
