## [1.0.2] — 2026-06-04

### Fixed
- **Generic Electron icon in the GNOME dock** (source/dev runs) — the app now names itself `hivemind`, so the X11 `WM_CLASS` matches its desktop entry and the real icon shows instead of the gear. Packaged AppImages already had the icon.
- **Upgrade-safe profile migration.** Naming the app moves its data dir to `~/.config/hivemind`; on first run with the new name the app now clones the most recent legacy profile (`Electron` for dev, `@hivemind/desktop` for packaged) into it, so **upgrading does not blank your canvas** (frames/tiles/layout). Idempotent. `apps/desktop/src/main/index.ts`.

## [1.0.1] — 2026-06-04

### Added
- **App icon.** hivemind now has a real icon — two stacked canvas frames holding a lit honeycomb (the canvas-of-frames + the hive), honey-amber on dark. Wired as the AppImage/desktop icon (`electron-builder` `linux.icon`), the live window/taskbar icon, and the renderer favicon; PNGs in `apps/desktop/icons/` + `assets/icon.png`. `apps/desktop/{icons/icon.png,package.json,src/main/index.ts,src/renderer/{index.html,public/*}}`, `.github/workflows/release.yml`.
- **Browser in the tool island.** The top-center tool island now has a **Browser** button (hotkey `7`) alongside terminal / agent / explorer / diff / issues / frame — the Browser tile was previously only reachable via the hidden `7` key or a frame's **+** menu. `apps/desktop/src/renderer/src/{canvas-islands,Canvas}.tsx`.

### Fixed
- **`hivemind upgrade` no longer opens the app** — it just upgrades. New installs intercept `upgrade` in the launcher, but an old launcher (or a bare AppImage symlink) passed the word straight to Electron, which ignored it and opened a window. The app binary now handles `upgrade` itself: runs the official installer, streams its output, and exits without creating a window. `apps/desktop/src/main/index.ts`.
- **Diff tile no longer crashes on remote frames** with `CodeView.addItem: duplicate id …`. A malformed/duplicate git path (seen over SSH) produced two CodeView items with the same id, which threw and took down the whole tile. Items are now de-duped by id (first wins, dev-warns) so a duplicate degrades instead of crashing. `apps/desktop/src/renderer/src/DiffTile.tsx`.
- **Diff tile listed untracked directories as empty `+0 -0` rows.** `git status` collapses an untracked directory into one `dir/` entry, which the diff rendered as an undiffable blank. Status now uses `--untracked-files=all` so each new file is a real new-file diff. `apps/desktop/src/main/git-adapter.ts`.

### Changed
- **Diff tile's changed-files sidebar now uses the `@pierre/trees` `<FileTree>`** — the same component the editor's file tree uses — instead of a bespoke tree. Deep paths (`java/com/flotomate/models/…`) get VS Code-style compact folders, file icons, virtualization, and `⌘P` quick-open; filenames stop truncating. Per-file +adds/−dels show as a row decoration; jump-to-file and toggle-reviewed live in the row context menu. `apps/desktop/src/renderer/src/DiffTile.tsx`.

The skill ships into agentic projects via `installAgenticStack` (write-if-absent, like `hive-work`) and uses `npx -y agent-browser` so there's no global install. Skill at `.claude/skills/hive-browser/SKILL.md`. Links/`target=_blank`/`window.open` open as **new tabs** in the same tile (canvas-native tabs) rather than rogue OS windows. Built on Electron's `<webview>` so it lives in the DOM and pans/zooms/clips with the react-flow canvas (a native `WebContentsView` overlay can't). Each tab owns a real `webContents`, so an agent can drive the **visible** tab over the Chrome DevTools Protocol via the new main-process bridge (`browserCdp` / `browser:register`) — navigate, click, read the DOM, screenshot, evaluate; a 🤖 toolbar button runs a live CDP self-test. Guests are hardened on attach (no node/preload). `apps/desktop/src/{main/index.ts,preload/index.ts,shared/ipc.ts,renderer/src/{BrowserTile.tsx,canvas-nodes,canvas-node-build,useSpawn,FrameNode,LayersPanel,tile-kinds,canvas-sizing,canvas-persistence}}`.

- **Codex + opencode agents.** A frame's **+** menu lists every enabled agent (Claude / Codex / opencode) with its icon; spawning runs that agent's CLI as an agent-terminal and live status detection keys off the command (`detectCodex`/`detectOpencode`). MVP: no deterministic CLI resume for codex/opencode yet; both use a generic mark until logos are bundled. `apps/desktop/src/renderer/src/{agents.tsx,useSpawn,FrameNode,LayersPanel,Canvas}.tsx`.

- **Editor works in non-git folders** — the file tree falls back to a filesystem walk when the folder isn't a git repo (git decorations just don't show).
- **Remote SSH frames now support password auth** (in addition to SSH agent + private key), incl. keyboard-interactive servers. The connect form has a password field; the password is held in memory only, never written to disk. `apps/desktop/src/{main/remote/conn.ts,renderer/src/components/RemoteConnectModal.tsx,shared/ipc.ts}`.
- **Remote SSH frames.** A canvas frame can now be bound to a directory on another machine over SSH — every tile inside it runs on the remote: the terminal/Claude is a real PTY on the host, the editor reads/writes files over SFTP, and diff/status shell out to `git` on the remote. A frame's "remote" button opens a connect form (host/user/port, SSH-agent or key auth) then an SFTP folder browser to pick the directory. ### Fixed
- **Terminal text no longer "grows" after a hivemind restart.** On re-attach the PTY daemon keeps the session's original (frozen) cols/rows; the renderer's init `fit()` had already set `term.cols`, so `term.onResize` never fired and the daemon never learned the real geometry — claude stayed stuck at the old (often smaller) size and rendered narrow/oversized inside the tile. The renderer now force-syncs the current `fit()` dimensions on every spawn/attach. `apps/desktop/src/renderer/src/TerminalTile.tsx`.
- **`Ctrl+R` no longer re-opens the `⌘K` command palette.** Recents were folded into the `⌘K` palette but `Ctrl+R` was still wired to "open recent", so both shortcuts opened the same panel. `Ctrl+R` is now swallowed (so a muscle-memory reload doesn't tear down the canvas + re-attach every PTY); recents stay in `⌘K` and File ▸ Open Recent. `apps/desktop/src/{main/index.ts,renderer/src/App.tsx}`.
- **Worktree nesting hardening (architecture + performance review).** Deleting a repo frame no longer orphans its worktree children (they're removed from the canvas; the worktrees stay on disk, detach `×` is the destructive path). A tile dropped inside a worktree frame now reliably joins *that* worktree (its PTY runs on the right branch/cwd) — `parentFrameOf` prefers the innermost frame. Spawning/dragging a worktree no longer makes the nest jump (the separation pass pins the anchor's parent). A worktree frame dragged out of its parent now **detaches** instead of snapping back. Guards against >2-level nesting and double-create races. Tiles and worktree sub-frames share one occupancy model so the parent divides space without overlap. `apps/desktop/src/renderer/src/{Canvas,frame-layout}.{ts,tsx}`.
- **Performance: the branch badge stays off the hot path.** It uses a dedicated long-`staleTime` `git:branch` query (HEAD-gated refresh), not `git:status` — so it doesn't refetch/re-render on every file write during agent work. The selection pipeline returns the node array verbatim when nothing is selected (baked tile zIndex), and the frame auto-fit geometry uses single-pass min/max + capped collision iterations + a consistent dead-band (no self-fire on sub-pixel residue). `apps/desktop/src/renderer/src/{queries,Canvas,frame-layout}.{ts,tsx}`.

## [0.3.2] — 2026-06-03

### Fixed
- **Terminal text selection (and clicks) landed on the wrong row when zoomed.** xterm maps the mouse to a cell using its *unscaled* cell size, but react-flow CSS-scales the canvas at any zoom ≠ 100% — so selecting a line grabbed one 2–3 rows off (a known xterm limitation under CSS transform; it also threw off link clicks and mouse-reporting clicks). Selecting a terminal tile now snaps the canvas to exactly 100%, where the mouse→cell math is pixel-accurate, so selection, link clicks, and mouse clicks all hit the right row. (Editor/diff tiles are unaffected — they use DOM coordinates that scale correctly.) `apps/desktop/src/renderer/src/Canvas.tsx`.
- **Clicking a URL in the terminal now opens it in the browser.** The terminal had no link handling at all — URLs were only openable via an agent's own mouse-reporting, which is unreliable. Added xterm's WebLinksAddon (plain `http(s)` URLs become clickable) plus a `linkHandler` for OSC 8 hyperlinks (which claude emits); a click routes through `window.open` → the existing `setWindowOpenHandler` → `shell.openExternal`, opening in the OS browser (http/https only, no popup). `apps/desktop/src/renderer/src/TerminalTile.tsx`, `package.json`.
- **Image paste into claude was broken (regression in 0.3.1).** The clipboard work in 0.3.1 added a Cmd/Ctrl+V handler that called `preventDefault()` and pasted clipboard **text only** — so an image on the clipboard was swallowed and never reached claude (which reads images from the system clipboard on paste). Paste is no longer intercepted at all; the copy shortcut stays. `apps/desktop/src/renderer/src/TerminalTile.tsx`.

## [0.3.1] — 2026-06-02

### Changed
- **Spawning honors the current selection instead of always asking.** With 2+ frames, opening a tile (claude/shell/diff/…) from the ToolIsland, palette, or hotkey used to always pop the workspace picker. Now a selected frame — or the frame holding the selected tile — IS the target, so it spawns straight in; the picker only appears when nothing is selected to disambiguate. `apps/desktop/src/renderer/src/Canvas.tsx`.
- **Crisper terminal text — especially when a tile is selected.** xterm rasterizes its glyphs to a canvas that the pan/zoom viewport then CSS-transforms; a fractional pan offset or an off-by-epsilon zoom lands that bitmap on sub-pixels and looks blurry, and it got *worse* on select because selecting promotes the tile to its own compositing layer (the known xyflow#3282 "viewport transform not rounded to integers" issue). The live viewport transform now snaps to whole device pixels (and a near-1 zoom to exactly 1.0) whenever it settles — after a pan, after a fling, and on tile select — so text is sharp at rest and selecting a tile no longer fuzzes it. (Zoom levels other than 100% still scale the bitmap — inherent to canvas content.) `apps/desktop/src/renderer/src/Canvas.tsx`.

### Changed
- **Sharper terminal glyphs (xterm rendering knobs).** Enabled `rescaleOverlappingGlyphs` (glyphs that overflow their cell — powerline, wide Unicode — are rescaled to fit instead of clipping/smearing) and made `customGlyphs` explicit (box-drawing/block characters drawn as crisp vectors, not font bitmaps). xterm 5.5 has no option to render above the display's device-pixel-ratio, so this plus the self-hosted font + atlas rebuild is the realistic ceiling for crispness. `apps/desktop/src/renderer/src/TerminalTile.tsx`.

### Fixed
- **Terminals had no copy/paste, and Ctrl/Cmd+wheel didn't zoom over a tile.** xterm doesn't copy on its own (Ctrl+C just sends SIGINT), so there was no way to copy terminal text. Added the standard rule: **Cmd/Ctrl(+Shift)+C copies the selection** (and clears it, so a second press still interrupts); plain Ctrl+C with nothing selected still sends SIGINT; **Cmd/Ctrl(+Shift)+V pastes** (bracketed-paste aware). Separately, **Ctrl/Cmd+wheel now zooms the canvas even when the cursor is over a selected tile** — it's handled in the capture phase so it beats the terminal's own scroll instead of being swallowed by it. `apps/desktop/src/renderer/src/{TerminalTile,Canvas}.tsx`.
- **Blurry / mis-weighted terminal & code text (wrong font).** The terminal font (JetBrains Mono) was loaded from the Google Fonts CDN via a `@import`, which is async — and xterm measures its font at open and builds a glyph atlas from whatever is available *then*. With the web font not yet loaded it built the atlas on the fallback font's metrics and never corrected, so terminals rendered slightly blurry/mis-spaced (and on a cold/offline launch the real font never arrived at all). JetBrains Mono is now **self-hosted** (bundled via `@fontsource`, offline-safe), and each terminal **rebuilds its glyph atlas once the font is actually loaded** (re-fit + clear the WebGL texture atlas + refresh), so glyphs render at the correct metrics. `apps/desktop/src/renderer/src/{main.tsx,TerminalTile.tsx,styles.css}`, `package.json`.
- **Diff / file tree showed gitignored files that were already tracked.** `.gitignore` only affects *untracked* files — a file committed before being added to `.gitignore` keeps showing in `git status`/`diff`, and plain `git check-ignore` reports a tracked path as "not ignored". The diff and file-tree now filter their file lists through `git check-ignore --no-index`, which applies the ignore rules regardless of tracking, so a gitignored-but-tracked file (e.g. a committed `config.json` you later added to `.gitignore`) no longer clutters the diff. Untracked-ignored files were already excluded; this closes the tracked case. `apps/desktop/src/main/git-adapter.ts`.
- **One crashing tile no longer blacks the entire app.** Tiles are arbitrary React (xterm, the pierre diff `CodeView`, CodeMirror) and any one can throw during render — with no error boundary a single throw unmounts the whole React tree, so the entire canvas goes black (hit in practice by a diff tile). Because the layout persists, that tile reopened on every launch and re-blacked the app. Each tile is now wrapped in a `TileErrorBoundary`: a crash is isolated to that one tile (shown as a readable fallback with the error + Retry/Close), the rest of the canvas keeps working, and live PTY sessions are untouched (they live in the daemon). `apps/desktop/src/renderer/src/TileErrorBoundary.tsx`, `Canvas.tsx`.

## [0.3.0] — 2026-06-02

### Added
- **Tiles show the agent's own session title, and notifications name the workspace.** Agents set the terminal window title (OSC 0/2) to a live summary of what they're doing — claude writes a task title like "Refactor auth". hivemind now reads it (via xterm's title event) and shows it as the tile's session name on the header, the canvas session chips, toasts, and OS notifications, instead of a generic "claude #2". A user rename still wins and persists; the agent title is a live, non-persisted overlay. OS notifications also include the **frame (workspace) name** so a background ping tells you *which* project's agent wants you. `apps/desktop/src/renderer/src/{TerminalTile,Canvas,agent-state}.tsx`, `src/main/agent-notify-core.ts`, `src/shared/ipc.ts`.
- **Native OS notifications when an agent needs you or finishes.** When the hivemind window is unfocused, a desktop notification fires the moment any agent tile (all 15 detected CLIs, not just claude) transitions to a needs-you state (permission/question/blocked) or finishes a task (working→idle). Clicking it restores + focuses the window and flies the canvas to that tile; the window also flashes its taskbar entry until you look. The notification is driven off the existing in-app agent-status bus — one state machine, two surfaces (in-app toast when the window is focused but the tile isn't selected; OS popup when you're away) — so it is transition-deduped and covers every agent. `apps/desktop/src/main/{agent-notify,agent-notify-core}.ts`, `src/renderer/src/Canvas.tsx`.

### Fixed
- **"▶ Work on this" often did nothing (or prompted the wrong agent).** The button spawned a fresh claude tile and then fired the work prompt on a blind `setTimeout(2500)` aimed at the "latest" claude. Two failures: with 2+ workspaces the spawn first opens a frame **picker** (no tile yet), so the 2500ms send landed on an *old* claude or vanished; and even single-frame, 2.5s routinely lost the race against claude+MCP startup, typing the prompt before the input was ready. The prompt is now **attached to the spawned tile** and the tile delivers it to itself the first moment it reaches a ready input prompt (one-shot, via the agent-status detection) — it survives the picker and never races startup. `apps/desktop/src/renderer/src/{claude-bus,Canvas,TerminalTile}.tsx`, `IssuesTile.tsx`, `components/IssuePeek.tsx`.
- **False/missed agent notifications.** Two detection bugs undermined the notifications above: (1) the per-tile agent-status poll skipped entirely while the window was hidden/minimized — exactly when an OS notification matters — so finishes/prompts went unnoticed when you were away; it now keeps scanning (cost stays bounded by the existing output-dirty gate). (2) Claude briefly shows its idle prompt *between* tool calls, which the screen scrape misread as "finished," firing a premature completion notification; a working→idle flip is now held until Claude has been genuinely quiet across a confirming scan. The previous claude-only `Notification`-hook log path (delayed, single-agent) is removed in favour of the unified bus. `apps/desktop/src/renderer/src/{TerminalTile,agent-state}.tsx`, `src/main/{claude-resume,pty-daemon}.ts`.
- **Scrolling inside a selected tile (claude / diff / editor) didn't work.** The per-tile wheel handler ran in the CAPTURE phase and called `stopPropagation()` for a selected tile — which stops the event before it ever reaches the terminal/editor, so xterm scrollback and claude's mouse-reporting scroll got nothing. The handler now binds in the BUBBLE phase when the tile is selected (content scrolls first, then react-flow is blocked from also panning) and only uses the capture phase when unselected (to intercept the wheel before the content and pan the canvas). Affects every tile type via the shared `useTileWheelZoom`. `apps/desktop/src/renderer/src/Canvas.tsx`.

## [0.2.0] — 2026-06-01

### Fixed
- **Spawning a tile in a new frame disturbed existing claude sessions.** Off-viewport node culling (`onlyRenderVisibleElements`, on past 8 nodes) UNMOUNTS culled nodes — and our tiles wrap live PTY sessions. Each spawn recenters the viewport onto the new tile, pushing existing claude tiles off-screen → they got culled → their PTY was torn down (detach + xterm/WebGL rebuild + replay, which reads as "the session restarted") in daemon mode, or outright killed and respawned (a genuinely NEW claude session) in the in-process path. Culling is now disabled — every terminal node stays mounted regardless of viewport (xterm's WebGL addon already falls back to the DOM renderer if the GPU context cap is hit). Regression e2e: 12 tiles stay mounted past the old cull threshold and across a hard pan. `apps/desktop/src/renderer/src/Canvas.tsx`.
- **claude session resume was unreliable with multiple frames.** Each claude tile records its live session id (to follow `/resume` switches) via a SessionStart hook. The old hook did an unsynchronized read-modify-write on a single shared `tile-sessions.json` through one fixed `.tmp` path — so when several frames' claude tiles fired their hooks at once on restart, writers lost each other's updates (reproduced: 7/24 survived) and a tile's tracked id vanished, breaking its resume. Tracking is now one file per tile (no shared state, unique tmp), so concurrent tiles can't clobber each other. The legacy map is still read as a migration fallback. `apps/desktop/src/main/tile-session-store.ts`.
- **A claude tile could VANISH when resuming a session whose history was gone.** When `claude --resume <id>` hit a missing JSONL, the daemon retries once with `--session-id` (fresh session, same id) so the tile survives. But the retry kills the old PTY and respawns in place, and the old PTY's kill-induced `exit` then ran the teardown path with no check that it was still the session's current PTY — deleting the just-retried live session. PTY `data`/`exit` handlers now ignore events from a PTY that's already been replaced. Caught by a new end-to-end integration test. `apps/desktop/src/main/pty-session-manager.ts`.

- **Upgrading over an old install left `hivemind` broken (self-exec launcher / `ENOENT`).** Two installer bugs compounded: (1) `write_launcher` did `cat > "$BIN_DIR/hivemind"` without first removing an existing launcher — and old installs left `~/.local/bin/hivemind` as a **symlink** to the extracted `AppRun`, so `cat >` followed the symlink and wrote the wrapper *into the genuine AppRun*, turning it into a script that execs itself (infinite loop, app never starts); (2) even with that fixed, the bundled AppRun auto-detects `APPDIR` by searching for `$1` as a file, but `$1` is `--no-sandbox`, so `APPDIR` collapsed to empty and it tried to exec `/hivemind` (`No such file or directory`). `write_launcher` now `rm -f`s the old launcher first (never writes through a symlink) and the generated wrapper exports `APPDIR` explicitly. The version-skip self-heal also detects an already-corrupted AppRun (missing the genuine `BIN=` marker) and re-extracts it from the AppImage, and always refreshes the launcher to the working format. Re-running `install.sh` now repairs machines the old installer broke. `install.sh`.

### Changed
- **Extracted the claude resume/tracking transforms into `claude-resume.ts`** (electron-free) so the resume logic is unit-testable. New integration test drives the real `SessionManager` + transforms + tracker `.cjs` with a faithful fake `claude`: two frames, `/resume` an old session in one + a new session in the other, snapshot, restart the daemon, and verify each tile resumes the correct session (incl. the `/resume` switch surviving restart) and that a missing-history resume recovers without losing the tile. `apps/desktop/src/main/{pty-daemon,claude-resume}.ts`, `apps/desktop/tests/unit/{tile-session-store,claude-resume.integration}.test.ts`.

### Added
- **Cross-repo issues — move, copy, and link issues across workspaces (and let agents manage multiple repos).** A new global **workspace registry** (`~/.config/hivemind/registry.json`, shared by the app, `hive` CLI, and MCP) maps each workspace prefix → its `.hivemind` root. Because an issue id like `MANAGEARK-3` already names its workspace via the prefix, a full id is its own cross-repo address — no extra qualifier. Workspaces auto-register on `hive init`, on opening/binding a project, and when the MCP server starts.
  - **Issue panel (IssuePeek): a Relations section** — link chips (click to open the linked issue in its own repo, × to unlink), a "+ Link issue" form (`relates`/`blocks`/`blocked-by`/`duplicates`/`parent-of`/`child-of`, reciprocal recorded automatically on the other end), and a **Transfer** control to **Copy** (source kept, both linked with `relates`) or **Move** (source deleted, new issue stamped `moved-from`) into another registered workspace. Move reopens the new issue in its destination.
  - **CLI:** `hive workspace list|register`, `hive move <ID> <PREFIX> [--copy]`, `hive relate <ID> <OTHER-ID> [--type ..] [--remove]`.
  - **MCP (agent multi-repo management):** every id-addressed tool is now cross-repo aware (an id whose prefix belongs to another registered repo resolves automatically), plus `hive_list_workspaces`, `hive_move_issue`, `hive_link_issue`, and a `workspace` filter on `hive_list_issues`. `hive_create_issue` gains `acceptance_criteria` so agents write structured criteria instead of cramming them into the description.
  - Move refuses an issue with sub-issues (would orphan their files). Files stay the source of truth; the registry is a lazily-pruned index. `packages/hive-core/{registry,cross-repo}.ts`, `packages/hive-mcp/src/index.ts`, `apps/cli/src/commands/{workspace,move,relate}.ts`, `apps/desktop/src/renderer/src/components/IssuePeek.tsx`.

### Fixed
- **Cross-repo issue leak — a frame bound to a repo with no `.hivemind` showed another repo's issues.** Issues/diff/tree tiles inside a workspace zone fall back to the canvas's base root only when the zone is a *worktree* (same repo, other branch). For a bound *workspace* zone whose repo has no `.hivemind`, the tile now scopes to `root: null` (empty board) instead of inheriting the launch repo's root — which had been rendering, e.g., the `manageark` board inside a frame labelled `motadata-itsm-server`. `apps/desktop/src/renderer/src/Canvas.tsx`.
- **`installAgentic` no longer throws for a repo without a workspace.** Binding/switching to a repo that has never been `hive init`'d fired a best-effort `installAgentic` that threw `no .hivemind workspace at …`, surfacing a noisy main-process handler error for an entirely expected state. It now returns `{ ok: false, reason: "no-workspace" }`. `apps/desktop/src/main/index.ts`.
- **Agent-written acceptance criteria now show in the Acceptance Criteria panel.** `hive_create_issue` only takes free-text `description`, so agents embed the checklist under a plain `Acceptance criteria:` label line (not a `## ` heading) inside the description — the parser only recognised a markdown heading, so those items stayed buried in the description and the dedicated panel showed empty. `parseSections` now also splits on a plain or bold `Acceptance criteria:` label; `serializeSections` rewrites it to a canonical heading on next save. `packages/hive-core/src/storage.ts`.

## [0.1.3] — 2026-06-01

### Performance
- **Agent status polls pause when the window is hidden.** Each active agent tile scans its screen every 1.2s to update its status dot; with `backgroundThrottling` off (so PTY output keeps flowing) those scans ran even while the window was minimized, with no UI to update. They now skip when `document.hidden`. `apps/desktop/src/renderer/src/TerminalTile.tsx`.
- **Heavy tiles load on demand (faster startup).** The diff tile (`@pierre/diffs` + its ~1.25MB WASM/cpp syntax engine) and the workbench editor (CodeMirror) are now `React.lazy` code-split, so a terminal-only session no longer pays to parse/compile them at launch — they fetch their chunk on first open (with a small "Loading…" placeholder). The WASM/cpp grammar chunks are fully out of the startup path now. (GPU accel is already tuned: async GPU-channel establishment, xterm WebGL renderer with DOM fallback, raised WebGL-context cap, no background throttling — no cargo-cult flags.) `apps/desktop/src/renderer/src/Canvas.tsx`.

### Added
- **Tracks each claude tile's LIVE session — resume follows `/resume` switches.** hivemind spawned `claude --session-id <uuid>` and resumed that fixed id, so if you switched the active session inside a tile (`/resume <other>`, `--continue`, or claude reassigning its id) a restart resumed the *original*, not what you were actually in. The daemon now injects a **merged** SessionStart hook (fires on start/resume/clear, doesn't clobber existing hooks like claude-mem) that records `tileId → live session_id` into `<userData>/tile-sessions.json`; restore prefers that tracked id (`--resume <live>`). The tile id is baked into the hook command so it's correct even with several claude tiles in one cwd. Proven end-to-end through the daemon. `apps/desktop/src/main/{pty-daemon,pty-session-manager}.ts`.

### Changed
- **Tiles are inert until selected (click to enter) — mouse AND keyboard.** A tile's content no longer responds to clicks or typing until you select the tile — previously you could type into a terminal or hit a button on a tile you hadn't selected. The content is locked via `pointer-events:none` (mouse), but that doesn't block the keyboard, so a focused xterm kept eating keystrokes after you deselected it — fixed by disabling xterm stdin + blurring when unselected, plus a global blur of any focused input (editor, fields) whose tile is deselected. Click a tile to select → its content goes live (terminal auto-focuses). The header drag-handle stays live so you can still move + select by the header, and an unselected tile's wheel still pans the canvas. Regression test in `keyboard-gate.spec.ts`. `apps/desktop/src/renderer/src/{Canvas.tsx,TerminalTile.tsx,styles.css}`.
- **Right/middle-mouse drag pans the canvas — even over a tile.** Two bugs: the native context menu popped on right-mouse-down and aborted the drag (suppressed now), AND react-flow's pane-pan only fires on the pane itself, so a tile (a node) captured the pointer and right/middle-drag did nothing once the cursor was over a tile (which, on a full canvas, is everywhere). The tile wrapper now drives the pan itself for middle/right-drag when the tile is unselected (a selected tile keeps middle/right for its own use, e.g. terminal middle-click paste). Space+drag still pans too. Regression tests in `rightdrag-pan.spec.ts`. Trade-off: no right-click menu inside the canvas. `apps/desktop/src/renderer/src/Canvas.tsx`.
- **Trackpad/wheel moves the canvas over tiles until you select one.** Heavy tiles ate the wheel (`.nowheel` + xterm/diff/editor have their own wheel listeners), so the board wouldn't pan whenever the cursor sat over a tile — you had to find empty space. A single native capture-phase wheel handler on each tile is now the authority: an **unselected** tile pans the canvas (blocking the content's own scroll); **click a tile to select it** and the wheel scrolls its content (xterm scrollback / diff / editor); click empty canvas to deselect and pan again. Ctrl/⌘ + wheel still zooms toward the cursor over any tile. Works for terminals too (xterm no longer swallows it when unselected). `apps/desktop/src/renderer/src/{Canvas,TerminalTile,DiffTile,WorkbenchTile,IssuesTile,EditorTile}.tsx`.

## [0.1.1] — 2026-05-28

### Fixed
- **Branch-diff no longer errors in repos without `origin/main`.** The diff tile's branch mode ran `git diff origin/main...HEAD` literally — repos that use `master`, haven't fetched `origin`, or have no remote hit `fatal: ambiguous argument 'origin/main...HEAD'` (exit 128) surfaced as a raw error. Now the base ref is resolved authoritatively from the remote's recorded default branch (`git symbolic-ref --short refs/remotes/origin/HEAD`) — no guessing-cascade through `main`/`master`/upstream (those return the wrong base in real repos). ## [0.1.0] — 2026-05-28

### Fixed
- **Spawn focus matches the "." focus-selected feel + no double-animation.** Two-stage: `setCenter` on the new tile's known coords immediately (instant, works before the node is DOM-measured), then once xyflow measures it, `fitView` to frame it nicely (zoom-to-fit, same as the `.` hotkey). `placeInFrame` is now the single focus+select authority for placed tiles — the extras-effect, shell-effect, `toggleTile`, and `frameOpen` no longer also focus (that fired the animation twice); they fall back to focusing only a LOOSE tile with no frame membership. `apps/desktop/src/renderer/src/Canvas.tsx`.
- **Spawning a tile now reliably centers the viewport on it.** `FocusOnTile` used `fitView({ nodes: [id] })`, which needs the node's **measured** DOM dimensions — a freshly-spawned (or culled off-screen) tile isn't measured yet, so fitView centered on a 0×0 box and did nothing. Now the focus path resolves the target's absolute center from our own `positions`/`sizes`/`frames` state and calls `setCenter` (keeping current zoom, clamped 0.5–1) — no dependency on xyflow having measured or rendered the node. `placeInFrame` centers on the exact coords it just assigned. `apps/desktop/src/renderer/src/Canvas.tsx`.
- **Resume retry now actually fires (it never did).** Two bugs kept the dead-tile recovery from working: (1) the retry was timing-gated on a 5s window, but a slow `SessionEnd` hook (e.g. claude-mem) delays claude's exit well past it; (2) more fundamentally, `restoreRetryTransform` was handed the *original* `--session-id` spec instead of the *effective* `--resume` spec, so it looked for `--resume`, found none, and returned null — a silent no-op. Now the retry is **output-driven**: a small rolling buffer scans the restored session's output for claude's "No conversation found" error and respawns with `--session-id <uuid>` (fresh session, same id) the instant it prints — killing the failed PTY, with a one-line banner. The transform now receives the effective spec. Timing-based exit retry kept as a fallback. Unit test added. `apps/desktop/src/main/pty-session-manager.ts`.
- **PTY session identity now binds to the TILE (and therefore the frame), not the workspace path.** `ptyId` was `hm:${cwd}:${tileId}` — so the moment a frame's `workspacePath` rebound (or the canvas re-derived `zoneRepo` at mount), the renderer asked the daemon for a different ptyId, orphaning the original session and spawning a fresh PTY at the new cwd. Now `ptyId = hm:${tileId}` (tileIds are already unique timestamps). Frame rebinding no longer breaks the session; the daemon's frozen spec keeps the ORIGINAL spawn cwd so claude resumes in the path its history is tied to. New tiles spawned after a rebind still pick up the new bound cwd. `apps/desktop/src/renderer/src/TerminalTile.tsx:44`. **Upgrade note:** existing daemon sessions keyed under the old cwd-bearing id will be orphaned once (daemon GCs idle); next spawn lands fresh.
- **Resume actually resumes again** — yesterday's switch to keeping `--session-id <uuid>` on restore broke resume (claude does NOT auto-resume on a bare `--session-id` when JSONL already exists). Restored the `--resume <uuid>` swap. To prevent that swap from dead-tiling the session when the JSONL is missing, `SessionManager` got `restoreRetryTransform` + `restoreRetryMs`: a restored session that exits non-zero within 5s respawns once with `--session-id <uuid>` (fresh session, same id) and prints a one-line yellow banner explaining the reset. Best of both: deterministic resume on happy path, no dead tile on miss, user sees what happened. `apps/desktop/src/main/pty-daemon.ts:137`, `apps/desktop/src/main/pty-session-manager.ts:249`.
- **Orphaned legacy PTY snapshots are now scrubbed at daemon boot.** Yesterday's `ptyId = hm:${tileId}` change left every snapshot from the old `hm:${cwd}:${tileId}` scheme unreachable — they'd reload into the frozen map on every daemon start and never attach to anything (memory leak that grew across reboots forever). `loadAllSnapshots()` now detects the legacy key shape (3+ colon-segments after `hm:`), drops the in-memory load, AND unlinks the file. One-shot migration. `apps/desktop/src/main/pty-daemon.ts:51`.
- **`TerminalTile` no longer re-runs its PTY effect on bare cwd changes in persistent mode.** With the tile-keyed ptyId, a cwd-only prop change would silently detach+re-attach to the same daemon session (which keeps its original spawn cwd in the frozen spec). Looked like "nothing happened" with no signal. Dep array is now `[ptyId, cmd]` in persistent mode (cwd is the daemon's job), `[ptyId, cwd, cmd]` in the legacy in-process path. `apps/desktop/src/renderer/src/TerminalTile.tsx:336`.
- **Claude tile no longer crashes on resume when the JSONL is missing.** `pty-daemon` swapped `--session-id <uuid>` → `--resume <uuid>` on snapshot restore so claude would loud-error on a missing session. That loud error killed the tile dead ("No conversation found with session ID: …" → exit 1) whenever claude's session storage had been wiped (config reset, signout/in, major-version upgrade, fresh machine with a surviving snapshot). Keep `--session-id <uuid>` on restore instead — claude treats it as idempotent (resume if JSONL exists, create-fresh-with-same-id if not). Same deterministic-id semantics, no crash. `apps/desktop/src/main/pty-daemon.ts:137`.

### Fixed (review pass)
- **`readConfig` self-heal race + non-atomic write.** Multiple IPC handlers (`createIssue`, `listIssues`, `resolveProject`, …) hit `readConfig` in parallel on first launch; when the file was invalid, every caller raced into `writeConfig` and a bare `fs.writeFile` could half-truncate. Now an in-process `Map<root, Promise<Config>>` coalesces concurrent reads to share one repair pass, and `writeConfig` writes to `config.yaml.<pid>.<ts>.tmp` then `fs.rename` (atomic). New test pins 20-way concurrent readConfig + verifies the file stays valid.
- **Editor `diffMode` map no longer leaks.** Closed tabs left their `diffMode[path]` entry behind; opening a stale path resurrected `true` and triggered an unwanted `gitFileContents` fetch. Tab-prune effect now drops `diffMode` entries alongside `meta` + `buffers`.
- **Activity timestamps no longer churn the file on first update.** `ActivityEntry` now carries `rawAt` (the on-disk form); serializer round-trips `rawAt ?? at`, so legacy `YYYY-MM-DD HH:MM` rows stay verbatim until a NEW entry is appended (the new one uses ISO-Z). Previously, the first `updateIssue` after upgrade rewrote every activity line and produced huge noisy diffs. New test pins zero churn.
- Dropped unused `getOriginalDoc` re-export in `EditorTile.tsx` (dead-import workaround).
- Editor + diff tile defaults now clamped to `window.innerWidth − 80` so 1400×900 doesn't spawn off-screen on 1366px laptop screens.

### Fixed
- **No more huge gap when spawning a tile in a frame.** The collision placement used a fixed column step of the widest-possible default tile (1400+px), so a new tile jumped a full tile-width to the right even when the existing tiles were narrower — leaving a big empty band. Now the new tile packs snug to the right of the rightmost existing tile's actual edge (+24px gap), top-aligned. Frame grows then `fitFrame` tightens.
- **Opening a tile in a frame no longer stacks it on top of an existing one.** `placeInFrame` used a fixed 1240×860 grid with `Math.round` bucketing; once tile defaults grew to 1400×900 a wide tile straddled two buckets, occupancy mis-detected, and the new tile landed on top of a sibling. Replaced with real rectangle collision: gather the actual boxes of every tile already in the frame, scan candidate slots left→right / top→down, place in the first that overlaps nothing, grow the frame to contain it, then `fitFrame` snaps it tight. New tile lands beside the others.
- **Frame `+` / color popovers render on top instead of behind tiles.** Frame nodes sit at zIndex ≤ 90 (below tiles); a child popover can't escape its parent's stacking context, so the "open in zone" menu and the color picker rendered BEHIND any tile inside the frame. Both now portal to `document.body` with fixed positioning anchored under their trigger button (z-9999) via a shared `AnchoredMenu` component with a click-away catcher.

### Changed
- **Frames now auto-size to their contents (architecture change).** Frame geometry is *derived* reactively from the bounding box of the tiles inside, via **explicit membership** (a persisted `frameOf` tile→frame map) — NOT geometry. A tile entering a frame grows it; moving a tile reflows it; the last tile leaving collapses the frame to a small labeled placeholder (still a visible, droppable workspace zone). Why explicit membership: a geometry-derived rule deadlocks — a freshly-spawned big tile whose center sits outside a collapsed placeholder frame would never be claimed, so the frame could never grow to it. `frameOf` breaks that loop entirely; geometry maps to membership in exactly one place (a tile's drop in `onNodeDragStop`), never in a render-reactive path.
  - **Membership set on drop/spawn**: a tile joins the frame whose rect contains its CENTER on drop, or becomes loose if dropped outside every frame. Spawning into a frame records membership directly.
  - **`extent: "parent"` dropped** — tiles are free to leave a frame (required for collapse-on-leave; `extent` previously clamped a parented tile *inside* so it could never be dragged out).
  - **Frames are no longer manually resizable** — `NodeResizer` + the ⤢ fit button removed; geometry is auto. Breathing room = the `FRAME_PAD` constant.
  - **Dragging a frame carries its tiles** — every `frameOf`-member's absolute position is translated by the frame delta on drag-stop.
  - **Migration**: old layouts (no `frameOf`) seed membership once from geometry on load.
  - Loop-safe: the auto-fit effect reads `frameOf` (not geometry) for membership, computes against `prev` frames in the updater, 2px dead-band, `frames` not a dependency. Fires only at human-action frequency.
  - `apps/desktop/src/renderer/src/Canvas.tsx` (`frameOf` state + persistence + migration, auto-fit effect, `mkTile`/`frameTiles`/`placeInFrame`/frame-drag all keyed on `frameOf`), `FrameNode.tsx` (NodeResizer + fit button removed).
- **Editor: inline diff toggle per tab (`@codemirror/merge` `unifiedMergeView`).** Each open tab now has a `⇄` button (active tab in the tab bar, plus a duplicate in the standalone tile header). Clicking it overlays the buffer's diff against `HEAD:<path>` in the SAME editor — deletions as widget rows, additions inline, per-chunk accept/reject controls. Toggle off returns to plain edit mode without rebuilding state (CodeMirror `Compartment.reconfigure`). Standalone `DiffTile` stays for browsing branch/commit diffs (where there's no editable buffer). Research: marimo-team/codemirror-ai, aziis98 review-tool walkthrough, Sourcegraph blob view — all use the same `unifiedMergeView` extension pattern instead of a separate diff view class.

### Fixed
- **Editor now focuses freshly-opened tabs.** Clicking a 2nd / 3rd file in the file tree appended a tab but kept focus on the first — `EditorTile`'s `active`-keepalive only re-targeted when the current tab was *removed*. Now we diff `tabs` against the previous render and switch `active` to any newly-added path (last wins if several land in one render).
- **Spawning a tile now always pans the viewport to it.** Previously the focus-on-spawn was gated on `nodes.length > 1`, so the very first tile of a session (or a tile that landed off the default viewport inside a frame) got selected but never centered — felt like "spawn did nothing." Applies to claude, extra terminals, shell, editor, diff, and issues — including the `+` menu inside frames.
- **Larger default sizes for Editor and Diff tiles.** Bumped from 760×520 / 720×460 → 1400×900, in line with the claude/terminal default. The old sizes truncated file lists and diff hunks badly on first open.
- **`FocusOnTile` no longer gates on DOM-measured dimensions.** Previously `fitView` was only called once xyflow had measured the new node's width/height — but a culled node (`onlyRenderVisibleElements` cuts off-screen nodes from the DOM) never gets measured, so the retry loop expired silently and the very first spawn of a session never centered. Now we call `fitView` immediately; xyflow's internal `batchContext.nodeQueue` queues the fit-view until the next node batch flushes (verified against `@xyflow/react/hooks/useReactFlow.ts`).
- **Diff/FileTree explicitly respect `.gitignore`.** Added `--ignored=no` to `git status` (already the default, made explicit so flag drift can't silently expose ignored files). `git ls-files --exclude-standard` already does the same for the file tree. **Note:** a file that was COMMITTED before being added to `.gitignore` will still show — that's how git works; `git rm --cached <path>` removes it from tracking.
- **Activity timestamps no longer drift by your timezone offset.** Activity rows were stored as `YYYY-MM-DD HH:MM` (UTC, but with no `Z` suffix). The renderer's `new Date(str)` parses that format as **local time** — so users in IST (+5:30) saw every newly-created activity row as "5h ago" instantly. Now stored as full ISO with `Z`. Parser still accepts the legacy format and normalizes it to ISO on read — old issue files auto-migrate on next write.
- **Pan (palm) works inside frames again.** xyflow auto-tags every draggable node with the internal `.nopan` class (see `@xyflow/system` d3-zoom filter), which blocks pan started from inside the node. Frames cover huge regions so pan / right-drag / Space+left-drag / rubber-band selection all died inside them. Fix: `.react-flow__node-frame { pointer-events: none }` so the browser routes the gesture to the `.react-flow__pane` container; header bar + resize handles opt back in via `pointer-events: auto`. Selecting a frame is via its header bar.
- **`readConfig` now self-heals broken `.hivemind/config.yaml`.** Configs missing `prefix` (early-init files, hand-edited workspaces) used to hard-throw `bad_config` from every IPC call — `createIssue`, `listIssues`, every UI action — surfacing as a raw stack trace. Now we fill defaults (prefix derived from repo dir name, `next_id: 1`, `agents: {}`) and rewrite the file once. Same path handles a missing config file entirely. Tests in `packages/hive-core/src/storage.test.ts` pin the behavior.
- **install.sh: AppImage now extracted instead of FUSE-mounted.** Ubuntu 22.04+ ships without `libfuse2`, which made the AppImage fail with `dlopen(): error loading libfuse.so.2` on every fresh install. Installer now runs `--appimage-extract` and symlinks the bundled `AppRun` — no FUSE, no sudo, works on all glibc Linux.

### Removed
- **Cycles** — dead feature ripped out: CLI `hive cycle` command, MCP `hive_list_cycles` tool, `Cycle` types, `listCycles`/`readCycle`/`writeCycle` storage funcs, `cycle` field on issue frontmatter, IPC `listCycles`, renderer `useCycles` hook. Never had any UI consuming it. Breaking change for any external script using `hive cycle …`.

## [0.0.1] — 2026-05-27

### Added
- First public OSS release.
- **CLI** (`hive`) — `init`, `init --agentic`, `new`, `mcp-stdio`, and friends. Single-binary distribution via `bun build --compile`.
- **Desktop app** — Electron + xyflow infinite canvas with terminal / diff / file-tree / editor / issues tiles, frame-as-workspace zones, persistent PTY daemon, Mosh-style replay (headless xterm + SerializeAddon), disk-backed snapshots, claude `--session-id` binding for deterministic resume across reboot.
- **MCP server** (`@hivemind/mcp`) — 9-tool stdio server wrapping `@hivemind/core` so claude can `get_issue`, `set_state`, `add_comment`, `mark_acceptance`, etc.
- **Templates** — per-workspace `.mcp.json` + `CLAUDE.md` + `.claude/skills/hive-work/SKILL.md` copied by `hive init --agentic`.
- **install.sh** — single script for both fresh install and in-place upgrade. Downloads prebuilt binaries from GitHub Releases by default; `--dev` flag clones and builds from source.
- **GitHub Actions** — `release.yml` (tag-driven build + publish on `v*.*.*`), `ci.yml` (typecheck + build + unit tests on every push / PR).
