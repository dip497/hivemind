## [Unreleased]

### Fixed
- **PTY session identity now binds to the TILE (and therefore the frame), not the workspace path.** `ptyId` was `hm:${cwd}:${tileId}` — so the moment a frame's `workspacePath` rebound (or the canvas re-derived `zoneRepo` at mount), the renderer asked the daemon for a different ptyId, orphaning the original session and spawning a fresh PTY at the new cwd. Now `ptyId = hm:${tileId}` (tileIds are already unique timestamps). Frame rebinding no longer breaks the session; the daemon's frozen spec keeps the ORIGINAL spawn cwd so claude resumes in the path its history is tied to. New tiles spawned after a rebind still pick up the new bound cwd. `apps/desktop/src/renderer/src/TerminalTile.tsx:44`. **Upgrade note:** existing daemon sessions keyed under the old cwd-bearing id will be orphaned once (daemon GCs idle); next spawn lands fresh.
- **Resume actually resumes again** — yesterday's switch to keeping `--session-id <uuid>` on restore broke resume (claude does NOT auto-resume on a bare `--session-id` when JSONL already exists). Restored the `--resume <uuid>` swap. To prevent that swap from dead-tiling the session when the JSONL is missing, `SessionManager` got `restoreRetryTransform` + `restoreRetryMs`: a restored session that exits non-zero within 5s respawns once with `--session-id <uuid>` (fresh session, same id) and prints a one-line yellow banner explaining the reset. Best of both: deterministic resume on happy path, no dead tile on miss, user sees what happened. `apps/desktop/src/main/pty-daemon.ts:137`, `apps/desktop/src/main/pty-session-manager.ts:249`.
- **Orphaned legacy PTY snapshots are now scrubbed at daemon boot.** Yesterday's `ptyId = hm:${tileId}` change left every snapshot from the old `hm:${cwd}:${tileId}` scheme unreachable — they'd reload into the frozen map on every daemon start and never attach to anything (memory leak that grew across reboots forever). `loadAllSnapshots()` now detects the legacy key shape (3+ colon-segments after `hm:`), drops the in-memory load, AND unlinks the file. One-shot migration. `apps/desktop/src/main/pty-daemon.ts:51`.
- **`TerminalTile` no longer re-runs its PTY effect on bare cwd changes in persistent mode.** With the tile-keyed ptyId, a cwd-only prop change would silently detach+re-attach to the same daemon session (which keeps its original spawn cwd in the frozen spec). Looked like "nothing happened" with no signal. Dep array is now `[ptyId, cmd]` in persistent mode (cwd is the daemon's job), `[ptyId, cwd, cmd]` in the legacy in-process path. `apps/desktop/src/renderer/src/TerminalTile.tsx:336`.
- **Claude tile no longer crashes on resume when the JSONL is missing.** `pty-daemon` swapped `--session-id <uuid>` → `--resume <uuid>` on snapshot restore so claude would loud-error on a missing session. That loud error killed the tile dead ("No conversation found with session ID: …" → exit 1) whenever claude's session storage had been wiped (config reset, signout/in, major-version upgrade, fresh machine with a surviving snapshot). Keep `--session-id <uuid>` on restore instead — claude treats it as idempotent (resume if JSONL exists, create-fresh-with-same-id if not). Same deterministic-id semantics, no crash. `apps/desktop/src/main/pty-daemon.ts:137`.

### Fixed (review pass)
- **`readConfig` self-heal race + non-atomic write.** Multiple IPC handlers (`createIssue`, `listIssues`, `resolveProject`, …) hit `readConfig` in parallel on first launch; when the file was invalid, every caller raced into `writeConfig` and a bare `fs.writeFile` could half-truncate. Now an in-process `Map<root, Promise<Config>>` coalesces concurrent reads to share one repair pass, and `writeConfig` writes to `config.yaml.<pid>.<ts>.tmp` then `fs.rename` (atomic). New test pins 20-way concurrent readConfig + verifies the file stays valid.
- **Editor `diffMode` map no longer leaks.** Closed tabs left their `diffMode[path]` entry behind; opening a stale path resurrected `true` and triggered an unwanted `gitFileContents` fetch. Tab-prune effect now drops `diffMode` entries alongside `meta` + `buffers`.
- **Activity timestamps no longer churn the file on first update.** `ActivityEntry` now carries `rawAt` (the on-disk form); serializer round-trips `rawAt ?? at`, so legacy `YYYY-MM-DD HH:MM` rows stay verbatim until a NEW entry is appended (the new one uses ISO-Z). Previously, the first `updateIssue` after upgrade rewrote every activity line and produced huge noisy diffs. New test pins zero churn.
- Dropped unused `getOriginalDoc` re-export in `EditorTile.tsx` (dead-import workaround).
- Editor + diff tile defaults now clamped to `window.innerWidth − 80` so 1400×900 doesn't spawn off-screen on 1366px laptop screens.

### Note
- **Frame selection: click the header to select.** With `pointer-events: none` on the frame body (the pan fix), clicking the dashed body region no longer selects the frame. Header bar remains the selection / drag handle.

### Fixed
- **Opening a tile in a frame no longer stacks it on top of an existing one.** `placeInFrame` used a fixed 1240×860 grid with `Math.round` bucketing; once tile defaults grew to 1400×900 a wide tile straddled two buckets, occupancy mis-detected, and the new tile landed on top of a sibling. Replaced with real rectangle collision: gather the actual boxes of every tile already in the frame, scan candidate slots left→right / top→down, place in the first that overlaps nothing, grow the frame to contain it, then `fitFrame` snaps it tight. New tile lands beside the others.
- **Frame `+` / color popovers render on top instead of behind tiles.** Frame nodes sit at zIndex ≤ 90 (below tiles); a child popover can't escape its parent's stacking context, so the "open in zone" menu and the color picker rendered BEHIND any tile inside the frame. Both now portal to `document.body` with fixed positioning anchored under their trigger button (z-9999) via a shared `AnchoredMenu` component with a click-away catcher.

### Added
- **Frame "fit to content" button.** A new ⤢ button in each frame's header reflows the frame to the bounding box of its child tiles — grows AND shrinks (the existing resize/place logic only ever auto-grew, so frames accumulated dead space). Moving the frame origin is safe: child relative positions recompute against the new origin on the next render, so tiles stay put while the frame snaps around them. `apps/desktop/src/renderer/src/Canvas.tsx` (`fitFrame`), `FrameNode.tsx` (header button).
- **Editor: inline diff toggle per tab (`@codemirror/merge` `unifiedMergeView`).** Each open tab now has a `⇄` button (active tab in the tab bar, plus a duplicate in the standalone tile header). Clicking it overlays the buffer's diff against `HEAD:<path>` in the SAME editor — deletions as widget rows, additions inline, per-chunk accept/reject controls. Toggle off returns to plain edit mode without rebuilding state (CodeMirror `Compartment.reconfigure`). Standalone `DiffTile` stays for browsing branch/commit diffs (where there's no editable buffer). Research: marimo-team/codemirror-ai, aziis98 review-tool walkthrough, Sourcegraph blob view — all use the same `unifiedMergeView` extension pattern instead of a separate diff view class.

### Fixed
- **Editor now focuses freshly-opened tabs.** Clicking a 2nd / 3rd file in the file tree appended a tab but kept focus on the first — `EditorTile`'s `active`-keepalive only re-targeted when the current tab was *removed*. Now we diff `tabs` against the previous render and switch `active` to any newly-added path (last wins if several land in one render).
- **Spawning a tile now always pans the viewport to it.** Previously the focus-on-spawn was gated on `nodes.length > 1`, so the very first tile of a session (or a tile that landed off the default viewport inside a frame) got selected but never centered — felt like "spawn did nothing." Applies to claude, extra terminals, shell, editor, diff, and issues — including the `+` menu inside frames.
- **Larger default sizes for Editor and Diff tiles.** Bumped from 760×520 / 720×460 → 1400×900, in line with the claude/terminal default. The old sizes truncated file lists and diff hunks badly on first open.
- **`FocusOnTile` no longer gates on DOM-measured dimensions.** Previously `fitView` was only called once xyflow had measured the new node's width/height — but a culled node (`onlyRenderVisibleElements` cuts off-screen nodes from the DOM) never gets measured, so the retry loop expired silently and the very first spawn of a session never centered. Now we call `fitView` immediately; xyflow's internal `batchContext.nodeQueue` queues the fit-view until the next node batch flushes (verified against `@xyflow/react/hooks/useReactFlow.ts`).
- **Diff/FileTree explicitly respect `.gitignore`.** Added `--ignored=no` to `git status` (already the default, made explicit so flag drift can't silently expose ignored files). `git ls-files --exclude-standard` already does the same for the file tree. **Note:** a file that was COMMITTED before being added to `.gitignore` will still show — that's how git works; `git rm --cached <path>` removes it from tracking.
- **Activity timestamps no longer drift by your timezone offset.** Activity rows were stored as `YYYY-MM-DD HH:MM` (UTC, but with no `Z` suffix). The renderer's `new Date(str)` parses that format as **local time** — so users in IST (+5:30) saw every newly-created activity row as "5h ago" instantly. Now stored as full ISO with `Z`. Parser still accepts the legacy format and normalizes it to ISO on read — old issue files auto-migrate on next write.
- **Pan (palm) works inside frames again.** xyflow auto-tags every draggable node with the internal `.nopan` class (see `@xyflow/system` d3-zoom filter), which blocks pan started from inside the node. Frames cover huge regions so pan / right-drag / Space+left-drag / rubber-band selection all died inside them. Fix: `.react-flow__node-frame { pointer-events: none }` so the browser routes the gesture to the `.react-flow__pane` container; header bar + resize handles opt back in via `pointer-events: auto`. Selecting a frame is via its header bar.
- **`readConfig` now self-heals broken `.hivemind/config.yaml`.** Configs missing `prefix` (early-init files, hand-edited workspaces) used to hard-throw `bad_config` from every IPC call — `createIssue`, `listIssues`, every UI action — surfacing as a raw stack trace. Now we fill defaults (prefix derived from repo dir name, `next_id: 1`, `agents: {}`) and rewrite the file once. Same path handles a missing config file entirely. Tests in `packages/hive-core/src/storage.test.ts` pin the behavior.
- **install.sh: AppImage now extracted instead of FUSE-mounted.** Ubuntu 22.04+ ships without `libfuse2`, which made the AppImage fail with `dlopen(): error loading libfuse.so.2` on every fresh install. Installer now runs `--appimage-extract` and symlinks the bundled `AppRun` — no FUSE, no sudo, works on all glibc Linux.

### Removed
- **Cycles** — dead feature ripped out: CLI `hive cycle` command, MCP `hive_list_cycles` tool, `Cycle` types, `listCycles`/`readCycle`/`writeCycle` storage funcs, `cycle` field on issue frontmatter, IPC `listCycles`, renderer `useCycles` hook. Never had any UI consuming it. Breaking change for any external script using `hive cycle …`.

## [0.0.1] — 2026-05-27

### Added
- First public OSS release.
- **CLI** (`hive`) — `init`, `init --agentic`, `new`, `mcp-stdio`, and friends. Single-binary distribution via `bun build --compile`.
- **Desktop app** — Electron + xyflow infinite canvas with terminal / diff / file-tree / editor / issues tiles, frame-as-workspace zones, persistent PTY daemon, Mosh-style replay (headless xterm + SerializeAddon), disk-backed snapshots, claude `--session-id` binding for deterministic resume across reboot.
- **MCP server** (`@hivemind/mcp`) — 9-tool stdio server wrapping `@hivemind/core` so claude can `get_issue`, `set_state`, `add_comment`, `mark_acceptance`, etc.
- **Templates** — per-workspace `.mcp.json` + `CLAUDE.md` + `.claude/skills/hive-work/SKILL.md` copied by `hive init --agentic`.
- **install.sh** — single script for both fresh install and in-place upgrade. Downloads prebuilt binaries from GitHub Releases by default; `--dev` flag clones and builds from source.
- **GitHub Actions** — `release.yml` (tag-driven build + publish on `v*.*.*`), `ci.yml` (typecheck + build + unit tests on every push / PR).
